# V10-9 第三章实验资产冻结说明

本文档冻结第三章实验入口、配置矩阵、输出目录和收敛性检查口径。V10-9 不强制一次性跑完整高射线数仿真，而是保证每个论文图表都有明确配置来源、输出位置和复现实验步骤。

## 1. 冻结资产

核心入口：

1. FineChannel 精细信道模板：`configs/app/v10_finechannel_template.json`
2. Coverage 覆盖模板：`configs/app/v10_coverage_template.json`
3. 实验矩阵脚本：`test/ch3_v10_freeze_manifest.py`
4. 已生成配置清单：`output/ch3_frozen/v10_ch3_manifest.json`
5. 已生成 case 配置目录：`output/ch3_frozen/configs/`

模板配置原则：

1. FineChannel 使用 `trace_profile=FineChannel`、确定性反射/透射分裂、动态 Rx 射线管半径、wedge tube coupling、Keller 锥采样、路径去重和几何残差过滤。
2. Coverage 使用 `trace_profile=Coverage`，保留动态 Rx 半径和 wedge tube coupling，但默认不保存逐路径细节，面向覆盖热图效率。
3. 天线文件均使用仓库相对路径，保证论文归档和复现实验时不依赖本机绝对路径。

## 2. 实验矩阵

运行：

```powershell
python test\ch3_v10_freeze_manifest.py --emit-configs
```

当前冻结 26 个 case：

| 实验组 | case 数 | 论文用途 |
|---|---:|---|
| `E1_antenna_model` | 4 | Ideal/dipole/patch/horn 天线模型对 APS、XPR、PDP、CIR 的影响 |
| `E2_pose_yaw` | 7 | patch 天线姿态 yaw 旋转对接收功率、APS、XPR 的影响 |
| `C1_ray_count_convergence` | 4 | `ray_count` 收敛性，证明 SBR 寻径采样充分性 |
| `C2_diffraction_sampling` | 4 | `diffraction_rays_per_event` 对绕射路径与 Keller residual 的敏感性 |
| `C3_rx_radius_sensitivity` | 4 | 接收球/射线管半径对漏径风险和命中率的影响 |
| `E3_coverage_heatmap` | 3 | 大规模覆盖热图和射线数收敛趋势 |

## 3. 输出目录规范

建议统一输出到：

```text
output/ch3_frozen/
  v10_ch3_manifest.json
  configs/<experiment_id>/<case_id>.json
  runs/<experiment_id>/<case_id>/
    config_snapshot.json
    paths/
    channel/
    coverage/
    diagnostics.json
    figures/
```

每个论文图表必须记录：

1. `experiment_id` 和 `case_id`
2. 使用的配置文件路径
3. RT 可执行文件版本或 git commit/status 摘要
4. `ray_count`、`trace_profile`、最大交互阶数、`diffraction_rays_per_event`
5. 天线 `pattern_file`、`polarization_file`、`forward/up`
6. 输出文件名和图表脚本名
7. 数据性质：仿真数据、解析验证数据、或外部测量数据

当前第三章实验均应标注为自研 RT 引擎仿真数据。若未引入实测数据，不应在论文中暗示为实测验证。

## 4. 收敛性与敏感性判据

`C1_ray_count_convergence`：

1. 比较总接收功率、有效路径数、主径功率、PDP 主峰位置。
2. 建议以最高射线数 case 作为参考，报告相对误差。
3. 若功率差异仍大于 1 dB，应继续提高 `ray_count` 或放宽/校正 Rx tube 半径。

`C2_diffraction_sampling`：

1. 比较绕射路径数、绕射功率占比、Keller residual 分布。
2. 若采样数增加后路径族仍明显变化，应在论文中说明绕射为近似采样机制，不宣称穷尽所有绕射径。

`C3_rx_radius_sensitivity`：

1. 比较路径命中率和总功率随 `rx_sphere_radius_m` 的变化。
2. 半径过大会引入虚假近邻路径，半径过小会漏径；论文中应以敏感性曲线说明选取依据。

`E1/E2/E3`：

1. 天线模型对比必须使用相同场景、Tx/Rx 位置、射线数和交互阶数。
2. 姿态扫描必须只改变 `forward/up`，不同时改变天线方向图、极化文件和 Rx 位置。
3. Coverage 热图用于覆盖趋势，不直接替代 FineChannel 的 APS/XPR/CIR 逐径分析。

## 5. 已知限制

1. 当前配置仍是单 `antenna` 块，Tx/Rx 不能在 JSON 中分别指定不同天线模型；若论文需要“Tx 固定、Rx 独立旋转”的严格实验，需要后续扩展配置结构。
2. Coverage 模板默认不导出逐路径细节，不适合直接计算精细 APS/XPR；精细信道指标应使用 FineChannel case。
3. 绕射路径仍是 Keller 锥采样近似，需用 `C2_diffraction_sampling` 说明采样数影响。
4. `validation/run_all.py --suite all` 当前仍有既有 critical skip：`G-6: Diffraction wedge`，因此 paper-ready 标记未完全转正；这不等于 V10-9 资产失败，但论文前建议补齐该验证。

## 6. V10-9 通过条件

1. `python test\ch3_v10_freeze_manifest.py` 能检查所有引用资产。
2. `python test\ch3_v10_freeze_manifest.py --emit-configs` 能生成 26 个 case 配置和 manifest。
3. 所有生成配置均可追溯到模板、case mutation 和预期输出。
4. 论文图表不得脱离 manifest 使用未登记配置或未标注 synthetic/仿真数据。
