# v10 开发手册：统一增强型 SBR 几何寻径重构

> 本文档用于指导 v10 阶段的几何寻径重构与后续自主开发。  
> 当前文档只作为开发手册和讨论基础，不代表已经进入代码实现。  
> 任何代码修改必须在用户审查并确认本手册后，再按本文档阶段顺序逐步实施。

## 1. v10 总目标

v10 的核心目标是重构当前 RT 引擎的几何寻径层，将现有割裂的 `Precise` 与 `Coverage/SBR` 两套寻径逻辑，逐步收敛为一个统一的、物理一致的增强型 SBR 正向追踪框架。

目标不是证明数学意义上的完备路径枚举，而是在给定以下约束条件时，生成稳定、可复现、可收敛验证的物理候选多径集合：

1. 发射射线角分辨率或射线管分辨率。
2. 最大总交互阶数。
3. 最大反射、透射、绕射次数。
4. 功率或复场幅度阈值。
5. Rx 接收球或射线管命中条件。
6. 路径去重和后验几何残差阈值。

v10 完成后，论文中应将寻径算法表述为：

> 本文采用统一增强型 SBR 正向射线追踪框架，通过确定性反射/透射分裂、射线管-楔边耦合的 UTD 绕射候选生成、接收球/射线管命中检测及路径后验校正，在给定角分辨率、交互阶数和功率阈值下生成物理一致的候选多径集合，并将其输入复矢量 EM 模块进行相干信道计算。

## 2. 当前问题基线

本节冻结 v10 前必须承认的现有问题，避免后续开发时误把旧行为当成正确基线。

### 2.1 Precise 模式问题

当前 `SearchEngine` 使用优先级队列和 expander 逐状态展开，入口见 `core/search/SearchEngine.cpp`。其主要问题是：

1. 初始状态方向直接设为 Tx 指向 Rx。
2. 反射 expander 使用单次镜像 Rx 的方法生成候选，适合单反，但不适合高阶反射序列。
3. 反射后 `current_direction` 被设为反射点指向 Rx，这会使中间状态过早闭合到 Rx。
4. 透射 expander 重新构造当前点到 Rx 的射线，而不是沿上一交互后的真实出射方向继续传播。
5. 绕射 expander 以当前点-候选边-Rx 做 Fermat 搜索，适合单次闭合候选，不适合作为统一正向多阶传播。
6. `per_expander_keep_limit`、`per_state_keep_limit` 使搜索具有启发式截断，不具备路径完备性保证。

结论：现有 precise 更适合称为“低阶候选路径搜索器”，不宜继续作为高阶混合路径的主几何引擎。

### 2.2 Coverage/SBR 模式问题

当前 `core/search/SbrEngine.cpp` 已具备正向射线追踪雏形，但主要定位仍是 coverage：

1. 透射与反射采用 Monte Carlo 分支选择，而不是确定性双分裂。
2. 连续两步无新 Rx 命中会提前终止，不适合少量 Rx 的精细信道寻径。
3. 部分路径记录存在每射线硬上限，可能漏掉有效多径。
4. 绕射仅在面元命中点靠近三角边时触发，属于简化边缘触发。
5. 当前绕射 Keller 锥采样数过低，且未形成与射线管半径、Rx 目标和功率阈值相关的自适应策略。
6. coverage 输出主要适合功率热图，不适合直接作为高保真 CIR/APS/XPR 的唯一依据。

结论：现有 SBR 是 v10 的最佳承接基础，但需要从 Monte Carlo coverage 升级为统一增强型 SBR。

## 3. v10 总体架构

v10 后目标架构如下：

```text
Scene preprocess
  faces / edges / wedges / BVH / wedge grid / Rx grid
        |
Unified SBR tracer
  Tx ray or ray-tube generation
  wavefront propagation
  face hit:
    reflection branch
    transmission branch
  wedge coupling:
    UTD Keller cone diffraction branches
  Rx hit:
    candidate path recording
  pruning:
    order / power / tube / Rx reachability / top-N / dedup
  optional path refinement:
    geometric residual correction
        |
GeometricPathSet
        |
EM solver
  Tx Jones injection
  free-space propagation
  Fresnel reflection/transmission
  UTD diffraction coefficient
  Rx conjugate matching
        |
Channel export
  paths / CIR / PDP / APS / XPR / coverage / ISAC
```

从用户视角仍可保留两个 profile，但不再对应两套寻径机制：

| Profile | 用途 | 寻径核心 | EM/导出 |
| --- | --- | --- | --- |
| `FineChannel` | 少量 Rx 精细信道 | 统一增强型 SBR，高射线密度，小接收球，完整路径记录 | 复矢量 EM，相干 CIR/PDP/APS/XPR |
| `Coverage` | 大规模覆盖热图 | 同一 SBR 核心，较大接收球，简化路径记录 | 可只累计功率，Rx 天线可简化 |
| `DebugValidation` | 单元/解析场景验证 | 小场景，高诊断输出，可禁用剪枝 | 输出几何残差、路径树和调试 JSON |

## 4. 开发原则

1. 不一次性推倒重写。
2. 先保留现有 SBR 可运行链路，再逐步替换关键行为。
3. 每个阶段必须有独立测试闭环。
4. 每个阶段通过后，必须同步更新本文档的“实施记录”章节。
5. 几何寻径只负责生成可信 `GeometricPath`，复矢量电磁计算仍交给 `core/em`。
6. 任何路径数增加都必须同时伴随剪枝、去重和诊断指标。
7. FineChannel 不追求覆盖吞吐，Coverage 不追求完整相干路径。
8. 论文可用结果必须来自可复现配置和可追踪输出。

## 5. 核心数据结构规划

### 5.1 TraceProfile

建议新增统一追踪配置对象，避免继续把 `path_search` 和 `sbr` 中的字段语义混用。

建议字段：

```cpp
enum class TraceProfileKind {
    FineChannel,
    Coverage,
    DebugValidation
};

struct UnifiedTraceProfile {
    TraceProfileKind kind;
    int ray_count;
    int max_depth;
    int max_reflection_count;
    int max_transmission_count;
    int max_diffraction_count;
    double min_power_linear;
    double tx_angular_spacing_rad;
    double base_rx_sphere_radius_m;
    bool deterministic_reflection_transmission_split;
    bool enable_ray_tube;
    bool enable_wedge_tube_coupling;
    bool enable_path_refinement;
    bool store_full_paths;
    int max_paths_per_rx;
    int max_branches_per_state;
    int diffraction_rays_per_event;
};
```

第一阶段可先不新增完整对象，允许从现有 `AppConfig::sbr` 与 `AppConfig::path_search` 映射；但 v10 完整版应形成统一 profile。

### 5.2 RayState

当前 `SbrRayState` 需要逐步升级。

建议新增字段：

```cpp
struct UnifiedRayState {
    Point3 origin;
    Vec3 direction;
    double power_linear;
    ComplexVec3 field_hint;        // 可选。FineChannel 可以只记录几何，EM 后处理精算。
    double path_length_m;
    double solid_angle_sr;
    double tube_radius_m;
    int depth;
    int reflection_count;
    int transmission_count;
    int diffraction_count;
    int current_medium_id;
    int last_face_id;
    int last_wedge_id;
    std::vector<PathNode> nodes;
};
```

### 5.3 RayTube

射线管不是必须一步到位，但 v10 中后期必须引入：

```cpp
struct RayTubeFootprint {
    double solid_angle_sr;
    double radius_at_1m;
    double RadiusAt(double distance_m) const;
};
```

基本关系：

```text
tube_radius(distance) ~= distance * angular_spacing / 2
```

也可加入下限和上限：

```text
tube_radius = clamp(distance * angular_spacing * k, min_radius, max_radius)
```

### 5.4 WedgeCouplingCandidate

绕射候选应从“面元命中点靠边”升级为“射线管-楔边耦合”。

建议结构：

```cpp
struct WedgeCouplingCandidate {
    int wedge_id;
    int source_edge_id;
    Point3 diffraction_point;
    Vec3 wedge_direction;
    double distance_ray_to_wedge;
    double tube_radius;
    double incident_angle_to_edge_rad;
    double coupling_score;
    bool visible_from_ray_origin;
};
```

## 6. 几何算法设计

### 6.1 Tx 射线发射

短期可沿用 Fibonacci sphere。

中期建议支持：

1. 全空间均匀采样。
2. 基于 Tx 天线方向图的重要性采样。
3. 面向少量 Rx 的目标区域加密采样。
4. 固定随机种子或确定性序列，保证实验复现。

FineChannel 下，射线权重应考虑 Tx 天线增益上界或发射方向权重；Coverage 下可保持均匀权重。

验收条件：

1. 相同配置重复运行，射线方向序列一致。
2. `ray_count` 增加时，角间距单调降低。
3. 输出 diagnostics 中记录 `ray_count`、平均角间距、估计射线管半径。

### 6.2 面元反射/透射确定性分裂

现有 SBR 透射分支是 Monte Carlo 选择，应改为确定性分裂：

```text
当射线命中可反射/可透射面元：
  若 reflection_enabled 且 reflection_count 未超限：
      生成反射分支
  若 transmission_enabled 且 transmission_count 未超限且非 TIR：
      生成透射分支
```

反射方向：

```text
d_r = d_i - 2 dot(d_i, n) n
```

透射方向：

```text
由 SnellRefractV2 或等价函数计算
```

分支功率：

```text
P_ref = P_in * R_power_bound
P_trans = P_in * T_power_bound
```

FineChannel 下也可以只用功率作为寻径剪枝估计，最终 EM 仍使用 `ApplyReflectionInteraction` 与 `ApplyTransmissionInteraction` 计算复矢量。

验收条件：

1. 单平面反射场景：反射方向满足反射定律，残差小于阈值。
2. 单介质界面透射场景：透射方向满足 Snell 残差阈值。
3. TIR 场景：透射分支不生成，反射分支保留。
4. 同一射线命中玻璃墙时，反射与透射分支同时入队。
5. 分支数、被剪枝数、TIR 数写入 diagnostics。

### 6.3 射线管-Rx 命中

当前接收球半径是固定参数。v10 应支持：

```text
effective_rx_radius = max(config_rx_radius, tube_radius_at_rx_distance)
```

FineChannel 中接收球要小，并且命中后必须做几何残差记录；Coverage 中接收球可较大，用于稳定覆盖统计。

验收条件：

1. LOS 场景中 Rx 位于射线路径附近时可命中。
2. Rx 偏离射线管半径之外时不命中。
3. 增加射线数时，FineChannel 的有效接收半径应下降或保持受控。
4. 每个 Rx 输出命中射线数、候选路径数、去重后路径数。

### 6.4 射线管-楔边耦合

不建议使用严格零宽射线与线段求交。应使用传播段与 wedge 线段的最近距离。

给定传播段：

```text
S(t) = O + t d, 0 <= t <= L
```

给定 wedge 线段：

```text
E(u) = A + u (B-A), 0 <= u <= 1
```

求最近点 `S(t*)` 与 `E(u*)`。若：

```text
distance(S(t*), E(u*)) <= tube_radius(t*)
```

则认为该射线管照亮该楔边，绕射点取 `E(u*)`。

还需要验证：

1. 绕射点在 wedge 有效长度内。
2. 当前点到绕射点无遮挡，忽略构成 wedge 的相邻面。
3. 入射方向与 wedge 方向非退化。
4. wedge `valid_for_utd == true`。
5. 当前路径未超过绕射次数预算。

验收条件：

1. 人工构造 ray segment 穿过 wedge 附近的场景，最近距离小于 tube radius 时生成候选。
2. 最近点落在线段外时不生成候选。
3. 被遮挡 wedge 不生成候选。
4. 退化 wedge 不生成候选。
5. diagnostics 输出候选 wedge 数、通过可见性数、最终绕射事件数。

### 6.5 Keller 锥绕射射线生成

设 wedge 单位方向为 `e`，入射方向为 `d_i`。绕射射线方向 `d_o` 满足：

```text
dot(d_o, e) = dot(d_i, e)
```

构造垂直于 `e` 的正交基 `p, q`，则：

```text
c = dot(d_i, e)
s = sqrt(max(0, 1 - c*c))
d_o(phi) = c * e + s * (cos(phi) * p + sin(phi) * q)
```

采样策略：

1. DebugValidation：固定 `N_d` 均匀采样，便于复现。
2. Coverage：较小 `N_d`，例如 4-8。
3. FineChannel：较大 `N_d`，例如 16-64。
4. 少量 Rx 时，可对 Keller 锥上指向 Rx 附近的角区间加密。

采样后筛选：

1. 方向不能射入构成 wedge 的实体内部。
2. 不能立即自交。
3. 功率估计不能低于阈值。
4. 可选：若 FineChannel 且该方向未来不可能覆盖任何 Rx，可降低优先级或剪枝。

验收条件：

1. 每条绕射方向与 wedge 的夹角等于入射方向与 wedge 的夹角，误差小于阈值。
2. 采样方向单位化。
3. `N_d` 增加时候选方向数增加，路径结果呈收敛趋势。
4. 绕射 PathNode 记录 wedge_id、diffraction_point、incident_direction、outgoing direction、Keller residual。

### 6.6 路径后验几何校正

FineChannel 下，接收球命中的路径不应直接作为精确 CIR 路径。需要后验校正或至少记录几何残差。

第一版可先实现残差诊断：

1. 最后一段到 Rx 的偏移距离。
2. 反射点的反射定律残差。
3. 透射点的 Snell 残差。
4. 绕射点的 Keller cone 残差。

第二版实现局部校正：

1. LOS：投影到真实 Rx。
2. 反射路径：对交互点做小范围优化，使入射/出射满足反射定律并闭合到 Rx。
3. 透射路径：保留 Snell 残差阈值。
4. 绕射路径：在 wedge 线段上优化绕射点，使 Keller 条件和路径长度条件更稳定。

验收条件：

1. 输出每条路径的 `geometry_residual`.
2. FineChannel 默认丢弃残差超过阈值的路径。
3. 单反解析场景校正后路径长度与镜像法结果一致。

## 7. 剪枝策略

v10 的剪枝必须分层，不能只靠一个功率阈值。

### 7.1 硬预算剪枝

1. `max_depth`
2. `max_reflection_count`
3. `max_transmission_count`
4. `max_diffraction_count`
5. `max_paths_per_rx`
6. `max_total_active_rays`

### 7.2 功率上界剪枝

每个分支维护 `power_linear`。若：

```text
power_linear < min_power_linear
```

则终止。

FineChannel 可使用较低阈值，Coverage 可使用较高阈值。

### 7.3 Rx 可达性剪枝

少量 Rx 时，对当前射线段或延长线到每个 Rx 的最小距离进行估计：

```text
if min_distance_to_all_rx > tube_radius + rx_margin:
    降低优先级或剪枝
```

该规则在 DebugValidation 初期应默认关闭，以免误杀路径。

### 7.4 Wedge 候选剪枝

1. 只查询传播段附近 wedge，而不是 origin 附近 wedge。
2. 限制每个射线状态最多触发的 wedge 数。
3. 按最近距离、可见性、wedge 长度、预计功率排序。
4. FineChannel 中对 Rx 相关 Keller 扇区加密，对无关扇区降采样。

### 7.5 去重与合并

路径去重应使用：

1. 交互类型序列。
2. face_id / wedge_id 序列。
3. 交互点量化坐标。
4. delay bin。
5. AoA/AoD bin。

Coverage 可合并功率；FineChannel 应保留复振幅路径，但可对几何高度相似路径保留高功率代表。

验收条件：

1. 去重前后路径数输出。
2. 不同射线命中同一物理路径时不会重复污染 PDP/CIR。
3. DebugValidation 可关闭去重以观察原始路径树。

2. 冻结 v10 的目标、边界和术语。

3. 已确认论文表述不再承诺严格完备枚举。

文档同步：

1. 在“实施记录”中追加用户确认日期与讨论结论。

### 阶段 V10-1：SBR 确定性反射/透射分裂

目标：

1. 在现有 SBR wavefront 中将透射 Monte Carlo 分支改为确定性双分裂。
2. 反射和透射分支同时入队。
3. 保留原 coverage 行为为兼容选项。

主要涉及文件：

1. `core/search/SbrEngine.cpp`
2. `core/search/SbrEngine.h`
3. `core/common/config/AppConfig.h`
4. `core/common/config/AppConfigJsonCodec.cpp`
5. `app/RtPipeline.cpp`

测试闭环：

1. 单反射 smoke case。
2. 单透射 smoke case。
3. 玻璃面反射+透射双分支 case。
4. TIR case。
5. 原有 validation 全量回归。

通过条件：

1. Coverage 原有输出不发生不可解释崩溃。
2. FineChannel 或新增配置下同一面元可同时生成 R/T 分支。
3. diagnostics 中记录 R/T 分支生成数和剪枝数。

### 阶段 V10-2：FineChannel profile 与路径记录解除硬限制

目标：

1. 引入统一 profile 概念或最小映射层。
2. FineChannel 下禁用连续两步无新 Rx 早停。
3. FineChannel 下取消每射线 8 路径硬上限，改为 per-Rx top-N 和功率阈值。
4. 完整保留 `GeometricPath` 节点。

测试闭环：

1. LOS + 单反 + 单透小场景输出路径节点完整。
2. FineChannel 与 Coverage 同场景输出路径数不同但均可解释。
3. 路径导出 JSON/CSV 不缺关键字段。

通过条件：

1. FineChannel 可作为 EM 输入。
2. Coverage 仍可生成覆盖结果。
3. 路径数有上限保护，不会失控。

### 阶段 V10-3：射线管基础与动态接收球

目标：

1. 每条射线携带角间距或 solid angle。
2. 根据传播距离估计 tube radius。
3. Rx 命中半径支持动态计算。
4. 输出 ray tube diagnostics。

测试闭环：

1. LOS 场景中 ray_count 增加时动态半径降低。
2. Rx 偏移实验中命中/不命中边界符合 tube radius。
3. 覆盖结果随 ray_count 增加呈收敛趋势。

通过条件：

1. 动态接收球可配置开启/关闭。
2. FineChannel 默认启用较严格半径。
3. Coverage 默认保留较稳定半径下限。

### 阶段 V10-4：射线管-楔边耦合查询

目标：

1. 新增传播段附近 wedge 查询能力。
2. 实现 segment-segment 最近距离。
3. 用 tube radius 判断 wedge coupling。
4. 替代或增强当前 `DetectEdgeHit`。

## 12. 风险清单

| 风险 | 影响 | 控制方式 |
| --- | --- | --- |
| 分支爆炸 | 运行时间和内存失控 | 功率阈值、top-N、max active rays、分层剪枝 |
| 绕射误触发 | 虚假路径增多 | 可见性、tube coupling、Keller residual、功率阈值 |
| 绕射漏触发 | NLOS 覆盖不足 | 段-wedge 查询、Rx 相关 Keller 扇区加密 |
| 接收球过大 | 相位和时延不准 | FineChannel 使用动态小半径和后验校正 |
| 接收球过小 | 漏径 | ray_count sweep 和收敛性分析 |
| 去重过强 | 真实近邻路径被合并 | DebugValidation 可关闭去重，阈值可配置 |
| 去重过弱 | PDP/CIR 污染 | path signature 和 delay/AoA bin 合并 |
| EM 与几何不一致 | 指标错误 | 路径 residual + EM 单元测试 |
