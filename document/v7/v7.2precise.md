## 第1章：整体架构

### 1.1 两阶段流水线

```
┌─────────────────────────────────────────────┐
│ 阶段1: 几何寻径 (SearchEngine)              │
│   输入: 场景几何 + Tx/Rx 位置 + 交互配置     │
│   输出: GeometricPath[] (几何有效路径集合)    │
│   原理: Image Method + 优先级队列            │
│   约束: 仅几何约束 (可见性、介质侧、重复面)   │
├─────────────────────────────────────────────┤
│ 阶段2: 电场计算 (EM 链路)                    │
│   输入: GeometricPath[] + 材质数据 + 天线    │
│   输出: EMPathResult[] (复振幅/时延/极化)    │
│   原理: Fresnel TE/TM + UTD + Jones 矢量     │
│   后处理: CIR/PDP/APS/信道统计/导出          │
└─────────────────────────────────────────────┘
```

**通俗理解**：阶段1 负责"射线从 Tx 出发，经过哪些面/棱，最终到达 Rx 的几何路线"；阶段2 负责"每条路线上的电磁波衰减了多少、相位变了多少、极化方向怎样"。

### 1.2 与传统射线追踪的对应

| 概念 | 通俗解释 | 代码位置 |
|------|---------|---------|
| 射线 | 电磁波在自由空间中沿直线传播 | ApplyFreeSpaceSegment |
| 反射 | 射线碰到墙面，按镜面反射规律弹走 | ReflectionExpander + ApplyReflectionInteraction |
| 透射 | 射线穿过墙体，方向按 Snell 定律偏折 | TransmissionExpander + ApplyTransmissionInteraction |
| 绕射 | 射线碰到棱边，在 Keller 锥面上产生新射线 | DiffractionExpander + ApplyDiffractionInteraction |
| 路径 | 从 Tx 到 Rx 的一条完整射线链 | GeometricPath |
| 场 | 电磁波的复振幅 + 极化 + 时延 | FieldAccumulator |

## 第2章：几何寻径（SearchEngine）

### 2.1 整体流程

```
SearchEngine::Run():
  1. 创建初始状态: Tx位置 + 全部交互预算 (R/T/D counts)
  2. 推入优先队列
  3. 循环出队:
     a. 检查此状态是否可继续展开 (交互预算>0)
     b. 尝试构建 LOS 路径 (当前点→Rx 直连, 无遮挡)
     c. 调用三个展开器: 反射/透射/绕射
     d. 每个展开器返回候选下一状态列表
     e. 合并候选 → 去重 → 推入队列
  4. 队列为空或超时 → 返回所有找到的 GeometricPath[]
  5. EM 阶段逐路径计算电场
```

**代码位置**: `core/search/SearchEngine.cpp:244-343`

### 2.2 搜索状态 (PathState)

```cpp
struct PathState {
    Point3 current_point;          // 当前射线位置
    Vec3  current_direction;       // 当前射线方向
    int   current_medium_id;       // 当前介质 (空气=0)
    int   path_depth;              // 已经过的交互次数
    int   remaining_reflections;   // 剩余反射预算
    int   remaining_transmissions; // 剩余透射预算
    int   remaining_diffractions;  // 剩余绕射预算
    int   remaining_total;         // 剩余总交互预算
    int   last_hit_face_id;        // 上次命中的面元 (-1 表示 Tx)
    int   last_hit_wedge_id;       // 上次命中的楔边 (-1 表示无)
    InteractionType last_interaction_type;
    std::vector<PathNode> traversed_nodes; // 已走过的路径节点
    uint64_t state_signature;      // 状态签名 (去重用)
};
```

**通俗解释**：PathState 是搜索过程中的一个"快照"，记录射线在哪里、朝哪走、还剩多少弹射次数、已经过哪些面。每展开一次交互，就产生一个新的 PathState。

### 2.3 优先队列

**优先级公式** (`ComputeStatePriority`, 第45-51行):
```
priority = accumulated_length + distToRx - 0.5 * path_depth
```

- `accumulated_length`：已走过的总路径长度
- `distToRx`：当前点到 Rx 的直线距离
- `-0.5 * depth`：深度奖励——越深的路径优先级数值越小，越早出队

**通俗解释**：这是一个"最佳优先搜索"。优先扩展距离短 + 深度深的路径。深度奖励让算法先探索多跳路径（反射+透射+绕射组合），避免浅层路径挤满队列。

**成熟对比**：无线通信 RT 通常用 A*（加 heuristic 估计），纯光学 RT 常用 Dijkstra（仅距离）。本系统的深度奖励是自创的，适用于室内多跳场景。

### 2.4 LOS 路径检查 (TryBuildLosPath)

```
TryBuildLosPath(context, state):
  1. 从 state.current_point 向 context.rx_point 发射线
  2. 查询可见性: IsVisible(start, end)
  3. 如果无遮挡 → 构造 LOS GeometricPath
  4. 路径签名去重 → 加入结果集
```

**通俗解释**：每次展开前，先检查"当前点能不能直接看到 Rx"。如果能，就是一条 LOS（或经过若干交互后最终到达 Rx 的）路径。

**代码位置**: `core/search/SearchEngine.cpp` 中 `TryBuildLosPath`

### 2.5 反射展开 (ReflectionExpander)

```
ExpandReflection(context, state):
  for each 候选面元 (BVH 空间过滤):
    1. 镜像法: 将 Rx 关于面元平面做镜像 → mirrorRx
    2. 从 state.current_point 向 mirrorRx 发射线
    3. 检查射线是否命中此面元 (face_id 匹配)
    4. 如果命中 → 创建新 PathState:
       - current_point = 命中点
       - current_direction = 反射方向 (镜面反射)
       - remaining_reflections -= 1
       - last_hit_face_id = face_id
       - traversed_nodes 追加反射节点
    5. 几何合法性检查 (GeometryValidity)
    6. 候选按距离排序，取前 K 个
```

**BVH 空间过滤** (`ReflectionExpander.cpp:131-132`):
```
extended_region = AABB(tx, rx) + margin
margin = |tx - rx| * 0.8
```
只搜索这个 AABB 内的面元——远处的面不可能是反射面。

**通俗解释**：镜像法是初中物理"平面镜成像"的 3D 版本。想象 Rx 在镜子里有一个"虚像"mirrorRx。如果从 Tx 向 mirrorRx 射一线，命中镜子表面——那条线就是反射路径。算法对每个候选面元都做这个检查。

**理论依据**: Balanis Ch.5; 标准 Image Method

**成熟对比**: Sionna RT 用相同方法 (`image_method.py`)。Wireless InSite 用 SBR + IM 混合。

### 2.6 透射展开 (TransmissionExpander)

```
ExpandTransmission(context, state):
  for each 候选面元 (BVH 空间过滤):
    1. 检查介质侧语义 (front_medium ≠ back_medium)
    2. Snell 折射: n1*sinθ1 = n2*sinθ2
    3. 如果全内反射 (sinθ2 > 1) → 拒绝候选 (v7.4 A7)
    4. 创建新 PathState:
       - current_direction = 折射方向
       - current_medium_id = medium_out
       - remaining_transmissions -= 1
       - traversed_nodes 追加透射节点 (含 medium_in/out)
    5. 几何合法性检查
```

**通俗解释**：射线打到玻璃/墙壁上，一部分能量反射，一部分穿过去。透射展开器负责找"穿过去"的路径。方向由 Snell 定律决定（光从空气进玻璃会偏折），材质数据库提供折射率 n。

**关键修正 (v7.4 B15)**: 展开器现在写入 `node.incident_direction`，EM 层计算 Fresnel 时使用入射方向而非透射方向。

**理论依据**: Born & Wolf §1.5; Balanis Ch.5

### 2.7 绕射展开 (DiffractionExpander)

```
ExpandDiffraction(context, state):
  for each 候选楔边 (空间查询):
    1. 黄金分割搜索: 在楔边上找 P 使 |Tx-P| + |P-Rx| 最小
    2. Keller 锥验证: |cos(入射角)| = |cos(出射角)|
    3. 创建新 PathState:
       - current_point = Fermat 最短路径点
       - incident_direction = 入射方向 (供 EM 用)
       - remaining_diffractions -= 1
       - traversed_nodes 追加绕射节点
    4. 几何合法性检查
```

**通俗解释**：射线碰到墙角/棱边，不像反射那样弹走，也不像透射那样穿透——它在棱边上"绕"过去，产生一个锥形的散射波。展开器在棱边上找使总路径最短的点（Fermat 原理），然后验证它满足 Keller 锥条件。

**理论依据**: Kouyoumjian & Pathak (1974); Balanis Ch.12

**成熟对比**: Wireless InSite 用相同的 UTD 实现。Sionna RT 用近似公式。

### 2.8 几何合法性 (GeometryValidity)

```
检查项:
  1. 同面不重复交互 (刚打到的面不能马上又打)
  2. 透射介质侧必须语义完整 (medium_in ≠ medium_out)
  3. 交互次数不超过预算
  4. 路径深度一致性 (depth == interaction_count)
```

### 2.9 签名去重 (State/Path Signature)

```
StateSignature = hash(
  last_interaction_type,  // 最后是什么交互
  last_hit_face_id,       // 最后打到的面
  last_hit_wedge_id,      // 最后打到的棱
  current_medium_id,      // 当前在什么介质里
  path_depth,             // 深度
  remaining budgets...    // 剩余预算
)

PathSignature = hash(每个节点的 face_id + interaction_type + medium_ids)
```

**通俗解释**：两个搜索状态如果"本质上相同"（同面、同预算、同深度），就只保留一个，避免重复搜索。路径签名为最终路径做同样的事。碰撞概率 ~5×10⁻¹¹。

**代码位置**: `StateSignatureBuilder.cpp`, `PathSignatureBuilder.cpp`

## 第3章：电场计算（EM 链路）

### 3.1 整体流程

```
SolveSinglePathEM(config, scene, path, result, materialDb):
  1. PreparePathForEM(path)
  2. InitializeTxField(input, field)
     → field = {振幅=1, 相位=0, 极化=Tx天线方向, λ=c0/f}
  3. for each 路径节点 (索引 1..N-1):
     a. ApplyFreeSpaceSegment(field, segment_length)
        → 累加距离、时延(-k0·n·d 相位)、介质衰减
     b. 根据交互类型:
        ApplyReflectionInteraction(field, node)
        ApplyTransmissionInteraction(field, node)
        ApplyDiffractionInteraction(field, node)
  4. FinalizeAtReceiver(field, path)
     → FSPL(λ/4πd) + 介质衰减 exp(-Σαd) + Rx天线增益
     → 输出 EMPathResult
```

**代码位置**: `app/RtRealChainRunner.cpp:49-110`

### 3.2 Tx 场初始化 (InitializeTxField)

```
InitializeTxField:
  1. field.wavelength_m = c0 / f          // 波长
  2. field.amplitude_real = 1.0           // 初始振幅 = 1+0j
  3. field.phase_rad = 0.0                // 初始相位 = 0
  4. field.polarization = Tx天线极化方向   // (0,1,0) = 垂直极化 Y-up
  5. field.current_refractive_index = 1.0 // 自由空间 n=1
  6. 如果天线方向图已加载: 按发射方向查增益
     amplitude *= sqrt(gain_linear)
```

**通俗解释**：发射机发出单位振幅的信号（1∠0），极化方向由天线决定（默认垂直极化）。波长 λ = c₀/f 用于后续相位和 FSPL 计算。

### 3.3 自由空间传播 (ApplyFreeSpaceSegment)

```
ApplyFreeSpaceSegment(field, d):
  1. total_length += d                        // 几何距离累加
  2. delay += d * n_eff / c0                  // 传播时延 (介质中慢一些)
  3. phase_rad -= 2π * d * n_eff / λ          // 负相位积累 (-k·d)
  4. media_attenuation += α * d               // 介质衰减累加 (Np)
  5. FSPL 延迟到 FinalizeAtReceiver 一次性施加  // 避免逐段累乘
```

**通俗解释**：波在自由空间（或介质中）传播 d 米。时延 = 距离/速度；相位变负（因为波向前走了）；介质衰减先累加，最后一起算。**FSPL (λ/4πd) 只在最后算一次**，不在中间每段都乘。

**理论依据**: Friis (1946); Balanis Ch.4

### 3.4 反射交互 (ApplyReflectionInteraction)

```
ApplyReflectionInteraction(field, node):
  1. 从 node.direction 恢复入射方向 kInc = Reflect(kOut, n)  // v7.4 B10
  2. cosθ_i = |Dot(kInc, n)|
  3. 查询面元材质 → ε_c = ε_r - jσ/(ωε₀)
  4. Fresnel 反射系数:
     Γ_TE = (cosθ_i - √(ε_c - sin²θ_i)) / (cosθ_i + √(ε_c - sin²θ_i))
     Γ_TM = (ε_c·cosθ_i - √(ε_c - sin²θ_i)) / (ε_c·cosθ_i + √(ε_c - sin²θ_i))
  5. Jones 矢量投影:
     A_TE_inc = A_inc * <Jones, ê_TE>
     A_TM_inc = A_inc * <Jones, ê_TM>
  6. 反射后:
     A_TE_ref = Γ_TE * A_TE_inc
     A_TM_ref = Γ_TM * A_TM_inc
  7. 功率 |E_ref| = √(|A_TE_ref|² + |A_TM_ref|²)
  8. Jones_ref = (A_TE_ref·ê_TE + A_TM_ref·ê_TM) / |E_ref|  (归一化)
```

**偏振基向量**:
```
ê_TE = normalize(kInc × n)     // 垂直于入射面 (s-偏振)
ê_TM = normalize(ê_TE × kInc)  // 在入射面内 (p-偏振)
```

**通俗解释**：波打到墙面上，TE 分量（垂直于入射面的电场）和 TM 分量（在入射面内）的反射系数不同（Γ_TE ≠ Γ_TM）。这导致反射后偏振状态改变。算法先把入射波分解成 TE/TM，各乘自己的反射系数，再合成回去。

**关键修正 (v7.4 B10)**: 之前用出射方向 kOut 分解入射场（错），现在用 Reflect(kOut,n) 恢复入射方向（对）。eTM 方向修正后，TM 分量误差从可达 100% 修正到 < 1%。

**理论依据**: Balanis Ch.5, Eq.(5-49); Born & Wolf §1.5

**成熟对比**: Sionna RT 做同样的事（TE/TM 分解 + Jones calculus）。不同之处：Sionna 用圆极化基，本系统用 TE/TM 线极化基——物理等价，坐标系不同。

### 3.5 透射交互 (ApplyTransmissionInteraction)

```
ApplyTransmissionInteraction(field, node):
  1. kInc = node.incident_direction  // v7.4 B15: 用入射方向
  2. ε_c = ε_r - jσ/(ωε₀)
  3. Fresnel 透射系数:
     τ_TE = 2cosθ_i / (cosθ_i + √(ε_c - sin²θ_i))
     τ_TM = 2√ε_c·cosθ_i / (ε_c·cosθ_i + √(ε_c - sin²θ_i))  // v5 D6-A 修正
  4. TE/TM 分量投影 → 乘系数 → 矢量重构 (同反射)
  5. 介质衰减: α = k₀ * Im(√ε_c)  // v7 H1 修正
  6. 折射率: n_eff = Re(√ε_c)      // v7 H2 修正
  7. 后续自由空间段用 n_eff 调整相位和时延
```

**通俗解释**：穿过墙体。TE/TM 各有一套透射系数。进入新介质后，传播速度变慢 (n 倍)，衰减变大 (α)。介质衰减和折射率在这里更新，后续的自由空间段自动使用新值。

**关键修正 (v7.4 B15)**: 之前用透射后方向（kOut）计算入射角 → Fresnel 系数用错了 cosθ，偏差 20%+。现在展开器存储了 `incident_direction`，EM 直接使用。

**理论依据**: Balanis Ch.5, Eq.(5-50); Born & Wolf §1.5.2

### 3.6 绕射交互 (ApplyDiffractionInteraction)

```
ApplyDiffractionInteraction(field, node):
  1. 构建边缘固定坐标系:
     ê_z = 楔边方向
     ê_x = 垂直于ê_z且在0-face内的方向 (用面法线叉乘)
     ê_y = ê_z × ê_x
  2. Keller 锥: cosβ₀ = |kOut·ê_z|
  3. 入射/出射方位角 φ', φ 从0-face测量
  4. n = (360°-楔角)/180° (外角归一化)
  5. 距离参数 L = s₁s₂sin²β₀/(s₁+s₂)  // v7 C3 修正
  6. UTD 绕射系数 D_soft, D_hard (8点 GL 数值积分)
  7. Soft/Hard 投影 (v7 C4 修正方向):
     ê_soft = normalize(kOut × ê_z)  // 垂直于衍射面
     ê_hard = normalize(ê_soft × kOut)  // 在衍射面内
  8. D_soft 作用于 ê_soft 上的电场分量
     D_hard 作用于 ê_hard 上的电场分量
```

**UTD 系数 D 的四个 cot 项** (K&P Table 1):
```
T₁ = cot((π + (φ-φ'))/(2n)) * F(kL·a⁺(φ-φ'))
T₂ = cot((π - (φ-φ'))/(2n)) * F(kL·a⁻(φ-φ'))
T₃ = cot((π + (φ+φ'))/(2n)) * F(kL·a⁺(φ+φ'))
T₄ = cot((π - (φ+φ'))/(2n)) * F(kL·a⁻(φ+φ'))

D_soft  = T₁+T₂-T₃-T₄ (Dirichlet 边界, TE)
D_hard  = T₁+T₂+T₃+T₄ (Neumann 边界, TM)
```

**Fresnel 过渡函数 F(x)** (K&P Eq.25-28):
```
F(x) = 2j√x·e^(jx) ∫_{√x}^{∞} e^(-jτ²) dτ

数值求解: 8点 Gauss-Legendre × 16区间 × 步长0.5
x<1e-8 → F=0;  x>10 → F≈1 + j/(2x)
```

**通俗解释**：这是整个系统中最复杂的电磁计算。UTD 绕射理论用四个 cot 函数 + 一个 Fresnel 积分来描述棱边的散射。F(x) 让绕射场在"阴影边界"处平滑过渡（GTD 在边界处不连续，UTD 修复了这个问题）。8 点 Guass-Legendre 是数值积分的工程实现——不解析解 F(x)，而是用采样点求和逼近。

**关键修正汇总**:
- v7 C2: 边缘坐标系从 0-face 法线构建（不用 kOut）
- v7 C3: L 参数加 sin²β₀ 因子
- v7 C4: Soft/Hard 方向与系数正确配对
- v7 B19: x>10 渐近展开

**理论依据**: Kouyoumjian & Pathak (1974); Balanis Ch.12

**成熟对比**: 多数商业软件用解析近似公式（如 Luebbers 有理近似）。本系统用 8 点 GL 数值积分——精度高出 2-3 dB，是学术贡献点。

### 3.7 接收端收敛 (FinalizeAtReceiver)

```
FinalizeAtReceiver(field, path):
  1. FSPL: A_fspl = λ / (4π * total_length)
     功率: P_fspl = (λ/4πd)²
  2. 介质衰减: A_media = exp(-Σ α_i·d_i)
  3. Rx 天线增益: gain_lin = 10^(dBi/10)
  4. 总幅度: A_total = A_fspl * A_media * sqrt(gain_lin)
  5. amplitude_real = field.amplitude_real * A_total
     amplitude_imag = field.amplitude_imag * A_total (=0, Jones矢量已编码相位)
  6. 极化复制 (Jones矢量保持)
  7. AoA/AoD 从路径首尾节点提取
  8. 元数据 (Tx/Rx ID, 透射介质ID等)
```

**通俗解释**：在接收端，把整个路径的"总账单"算出来——自由空间损耗了多少（λ/4πd），介质里衰减了多少（e^(-αd)），Rx 天线增益了多少。然后输出最终结果。

**远场假设**: d > 2D²/λ ≈ 4m @ 2.4GHz。室内路径多数满足，短 LOS 路径 (< 4m) 近似成立。

**理论依据**: Friis (1946)

## 第5章：与成熟实现的对比

### 5.1 Sionna RT (NVIDIA)

### 5.2 Wireless InSite (Remcom)

| 维度 | Wireless InSite | 本系统 precise |
|------|----------------|---------------|
| 寻径 | SBR + IM 混合 | 纯 IM + 优先队列 |
| 绕射 | UTD (解析近似) | UTD (数值积分) |
| 漫散射 | 支持 (ER model) | 未实现 |
| 穿透深度 | 支持 | 未实现 |

### 5.3 本系统的学术优势

1. **UTD 绕射精度**: 8 点 GL 数值积分 > 解析近似 (2-3 dB)
2. **混合路径**: 反射+透射+绕射 任意组合 (Sionna 仅 IM 反射)
3. **Jones 矢量**: 完整复极化 (对标论文级实现)
4. **零外部依赖**: C++20 STL only

### 5.4 已知差距

1. **漫散射**: 未实现 → 多径尾部偏短
2. **穿透深度**: 面元无厚度 → 介质内路径长度不精确
3. **路径数**: IM 对 >1000 面场景组合爆炸 → 需引入 SBR
4. **时变**: 不支持 → 无法仿真移动性

## 第6章：面向未来的改进

### 6.1 几何层与 EM 层解耦

```
当前: ApplyReflectionInteraction 中有
  if (matName.empty()) return false;         // ← 几何信息, 不应在 EM
  if (!input.material_db) return false;      // ← 同上

改进:
  几何层: 放宽约束, 全量输出路径
  EM 层: 材质不可用 → 用真空 Fresnel → |Γ|=0 → 功率自然=0
         不需要 return false
```

### 6.2 EM 阶段物理剔除

```
路径在 EM 阶段被自然淘汰的场景:
  - 材质 ε_r=1 → Γ=0 → 反射功率=0 → 该路径贡献≈0
  - 功率 < 接收机灵敏度 → 标记为无效
  - 透射介质侧错误 → 功率自然归零
```

**收益**: 几何层找到更多路径 → EM 层自然淘汰 → 不会丢失有效路径
