# 05 — 验证与交叉对比框架

> 目的：建立理论↔仿真↔实测↔RT仿真的四方对比闭环，使每一级的结果差异可追溯到物理原因  
> 核心方法：差异溯源表（Discrepancy Attribution Matrix）

---

## 5.1 四方对比矩阵

```
                         数据源
           ┌──────────────┼──────────────┐
           ▼              ▼              ▼
        理论解         全波仿真         实测
     (解析公式)    (HFSS/CST FEM)   (暗室/VNA)
           │              │              │
           └──────────────┼──────────────┘
                          ▼
                    H2hRT RT 仿真
                          │
                          ▼
              信道指标 (PDP/APS/XPR/K/RMS)
                          │
                          ▼
         ┌────────────────┼────────────────┐
         ▼                ▼                ▼
    理论→RT           仿真→RT          实测→RT
    (基准)          (仿真验证)        (真实验证)
```

**每一列代表一种天线数据源驱动 RT 仿真后的信道输出。**

---

## 5.2 对比指标体系

### 5.2.1 方向图级对比（RT 仿真前）

| 对比项 | 方法 | 判据 |
|---|---|---|
| boresight 增益 | 三种数据源的值对比 | |ΔG| < 0.5 dB（理论 vs 仿真）、< 1.0 dB（仿真 vs 实测） |
| 3 dB 波束宽度 | 三种数据源的 HPBW 对比 | |ΔHPBW| < 3° |
| 第一旁瓣电平 | 仿真 vs 实测 | |ΔSLL| < 2 dB |
| 交叉极化比 (XPD) | boresight 处 co-pol/cross-pol | |ΔXPD| < 3 dB |
| 方向图相关性 | 二维归一化互相关系数 | ρ > 0.95（同一类型、不同来源） |

### 5.2.2 信道指标级对比（RT 仿真后）

| 对比项 | 关注差异 |
|---|---|
| 总接收功率 (dBm) | 来源于方向图 boresight 和平均增益差异 |
| RMS 延迟扩展 (ns) | 来源于方向图对不同延迟路径的选择性差异 |
| K-factor (dB) | 来源于方向图对 LOS vs NLOS 增益比的差异 |
| XPR mean (dB) | 来源于极化方向图的差异 |
| 2D APS 相关性 | 方向图差异的空间分布如何影响到达角分布 |

## 5.3 差异溯源表（核心方法论）

### 5.3.1 模板

当两个数据源（如仿真 vs 实测）的 RT 信道结果出现差异时，使用以下溯源表：

```
┌─────────────────────────────────────────────────────────────┐
│ 差异溯源报告                                                │
├──────────┬──────────────────────────────────────────────────┤
│ 对比对   │ 仿真(HFSS) vs 实测(方案B)                         │
│ 天线     │ LB-75-10-C-NF @ 12.5 GHz                         │
│ 信道指标 │ 总接收功率：仿真 −52.4 dBm vs 实测 −51.8 dBm     │
│ 差异     │ +0.6 dB (实测略高)                                │
├──────────┼──────────────────────────────────────────────────┤
│ 溯源步骤 │                                                  │
│ 1. 方向图│ boresight 增益：仿真 15.1 dBi vs 实测 15.3 dBi   │
│   差异   │ → +0.2 dB 贡献                                   │
│ 2. 波束  │ 仿真 HPBW=22° vs 实测 HPBW=24°                   │
│   宽度   │ → 实测在 10° off-boresight 处功率 −0.3 dB        │
│          │   该角度贡献 ~5% 总功率 → +0.15 dB 贡献           │
│ 3. 旁瓣  │ 仿真 SLL=−23 dB vs 实测 SLL=−25 dB               │
│   差异   │ → 旁瓣区域功率低 2 dB                             │
│          │   旁瓣贡献 ~2% 总功率 → −0.04 dB 贡献             │
│ 4. 残留  │ +0.6 − (+0.2 + 0.15 − 0.04) = +0.29 dB          │
│          │ → 归因于：测量阻抗失配 (0.15 dB) +               │
│          │   仿真材料损耗参数偏差 (0.1 dB) + 未建模因素      │
├──────────┼──────────────────────────────────────────────────┤
│ 结论     │ 差异的 60% 由方向图主瓣差异解释；                 │
│          │ 30% 为测量系统误差；10% 为建模不确定度。           │
│          │ 该差异在方案 B 的精度范围（±0.7 dB）内。          │
└──────────┴──────────────────────────────────────────────────┘
```

### 5.3.2 溯源步骤

1. **方向图对比**：逐角度计算 ΔG(θ,φ) = G_source1(θ,φ) − G_source2(θ,φ)
2. **加权映射**：用 RT 的 APS（角功率谱）作为加权函数，计算加权方向图偏差：
   $$
   \overline{\Delta G} = \frac{\iint \Delta G(\theta,\phi) \cdot P_{\text{APS}}(\theta,\phi) \cdot \sin\theta \, d\theta d\phi}{\iint P_{\text{APS}}(\theta,\phi) \cdot \sin\theta \, d\theta d\phi}
   $$
3. **分解到角度区域**：将 ΔG(θ,φ) 分解到主瓣区、旁瓣区、背瓣区，分别计算贡献
4. **信道指标映射**：总功率差 ≈ ΔḠ；RMS 差 ≈ 由 ΔG 在不同延迟路径上的选择性引起

## 5.4 各对比环的判据与行动

### 5.4.1 理论 vs 仿真

| 结果 | 行动 |
|---|---|
| |ΔG| < 0.3 dB @ boresight | ✅ 仿真通过验证，进入下一环 |
| 0.3 < |ΔG| < 1.0 dB | ⚠️ 检查网格收敛、PML 距离、端口设置 |
| |ΔG| > 1.0 dB | ❌ 仿真设置疑似错误，重新建模 |

### 5.4.2 仿真 vs 实测

| 结果 | 行动 |
|---|---|
| |ΔG| < 0.5 dB @ boresight | ✅ 仿真可替代实测（论文结论之一） |
| 0.5 < |ΔG| < 1.5 dB | ⚠️ 检查实测误差源（对准、多径、失配） |
| |ΔG| > 1.5 dB | ❌ 需复测或检查天线实物是否与仿真模型一致 |

### 5.4.3 理论/仿真/实测 → RT 信道指标

| 结果 | 含义 |
|---|---|
| 不同数据源的 RT 结果差异 < 测量不确定度 | 天线建模精度在此场景下对信道指标不敏感 |
| 差异 > 测量不确定度且可溯源到方向图特定区域 | **论文核心证据**：天线精细化建模对信道有可量化影响 |
| 差异 > 测量不确定度但无法溯源 | 其他因素（几何模型误差、材料参数等）在起作用 |

## 5.5 验证执行流程

```
1. 获取三种数据源的 CSV
2. 运行 CSV 验证脚本（检查格式、网格、周期边界）
3. 方向图级对比：
   a. boresight 增益
   b. 二维互相关系数
   c. 截面方向图叠加（E面/H面）
4. 导入 H2hRT：
   a. 使用同一场景、同一 SBR 参数、同一 Rx 位置
   b. 只改变 antenna_file 指向不同 CSV
5. 信道指标对比：
   a. 提取 channel_stats.json 关键指标
   b. 填充差异溯源表
6. 形成验证报告
```
