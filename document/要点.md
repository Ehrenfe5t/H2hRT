## 三、电磁理论核心（面试重点）

### 3.1 Fresnel 反射/透射

```
ε_c(f) = ε_r(f) - j·σ(f)/(ω·ε₀)    ← 复介电常数（e^{-jωt} 时谐）

Γ_TE = (cosθ_i - √(ε_c - sin²θ_i)) / (cosθ_i + √(ε_c - sin²θ_i))
Γ_TM = (ε_c·cosθ_i - √(ε_c - sin²θ_i)) / (ε_c·cosθ_i + √(ε_c - sin²θ_i))

T_TE = 2·cosθ_i / (cosθ_i + √(ε_c - sin²θ_i))
T_TM = 2·√ε_c·cosθ_i / (ε_c·cosθ_i + √(ε_c - sin²θ_i))
```

- Γ_TE ≠ Γ_TM → 反射后 TE/TM 分量相位差 → 椭圆极化
- 功率守恒约束：|Γ|² + (n_t·cosθ_t / n_i·cosθ_i)·|T|² = 1
- **自修复的 Bug**：TM 透射分子原写为 ε_c，应为 √ε_c。发现后修正并新增 L1 功率守恒验证项

### 3.2 UTD 一致性绕射理论（核心学术贡献）

**Kouyoumjian-Pathak (1974) 完整实现**：

```
D = -e^{-jπ/4} / (2n√(2πk)·sinβ₀) · [T₁ + T₂ ∓ T₃ ∓ T₄]

T₁ = cot((π + (φ-φ'))/(2n)) · F(kL·a⁺(φ-φ'))
T₂ = cot((π - (φ-φ'))/(2n)) · F(kL·a⁻(φ-φ'))
T₃ = cot((π + (φ+φ'))/(2n)) · F(kL·a⁺(φ+φ'))
T₄ = cot((π - (φ+φ'))/(2n)) · F(kL·a⁻(φ+φ'))
```

**Fresnel 过渡函数 F(x)**：
```
F(x) = 2j√x·e^{jx} · ∫_{√x}^{∞} e^{-jτ²} dτ
```
- 8-point Gauss-Legendre × 16 区间数值积分（非 Luebbers 有理近似）
- 128 采样点覆盖 8 rad 积分域，每振荡周期 ≥ 6.4 pts > Nyquist(2)
- 阴影边界过渡区误差 < 0.1%，优于工程常用近似（2-3 dB 误差）

**边缘坐标系**：ê_z = 楔边方向，ê_x = Cross(n_0face, ê_z)（从楔面法线构建），ê_y = ê_z × ê_x。φ, φ' 均从 0-face 测量——φ+φ' 在参考系旋转下不守恒，必须对齐楔面。

**Keller 锥**：cosβ₀ = |k̂·ê_z|，绕射射线在锥面上。距离参数 L = s₁·s₂·sin²β₀/(s₁+s₂)。

### 3.3 Jones 矢量极化

```
E = A · (p_re + j·p_im)    A∈ℂ 幅值, p∈ℂ³ 单位 Jones 矢量

TE/TM 复投影（非实投影）：
A_TE = A · ⟨p_re+j·p_im, e_TE⟩
A_TM = A · ⟨p_re+j·p_im, e_TM⟩

正交矢量重构（非标量加法）：
|A_new| = √(|A_TE|² + |A_TM|²)  ← 正交分量功率和
Jones_new = (A_TE·e_TE + A_TM·e_TM) / |A_new|
```

- **关键**：TE ⟂ TM 正交，当做标量加法 A_TE+A_TM 将导致正交分量对消——已在 v7 修复
- ISAC 交叉极化鉴别率 (XPD) 需要完整 Jones 矢量 → Stokes 参数不够

### 3.4 FSPL 与介质衰减

```
A_fspl = λ/(4πd)     Friis 场幅形式，仅施加一次（路径末端）
A_media = exp(-Σ α_i·d_i)     α_i 为各段介质衰减常数 (Np/m)
```

## 四、双模式寻径引擎

### 4.1 Precise 模式：Image Method + 优先级队列

```
算法流程：
1. 初始状态入优先队列 {Tx位置, 交互预算(R,T,D), 空路径签名}
2. 循环出队：
   a. 交互展开器展开候选路径
      - ReflectionExpander: 镜像法验证（虚Rx→镜像面→实Tx）
      - TransmissionExpander: Snell折射方向 + 介质侧语义检查
      - DiffractionExpander: Fermat黄金分割搜索 + Keller锥验证
   b. uint64 FNV-1a 哈希签名去重
   c. 几何合法性检查（同面不重复交互等）
   d. 到达Rx的路径纳入结果
3. 终止条件：队列空 或 预算耗尽
```

- 复杂度 O(B^D)，B=候选面元数，D=深度。签名去重砍掉 >99.9% 重复状态
- 室内场景 ~400 面，R4+T4+D2 depth=8 → 664 条路径

### 4.2 SBR Coverage 模式：Monte Carlo 射线追踪

```
射线循环：
  While (depth<max AND P>threshold):
    1. BVH射线→三角面求交 (Möller-Trumbore)
    2. RxHashGrid 接收球检测
    3. 绕射：Keller锥4方向 + 三因子UTD功率模型
    4. 透射：Monte Carlo概率选择(不分裂)
    5. 反射：Fresnel |Γ|²功率衰减
```

- Fibonacci 球面射线生成：φ_gold = π(3-√5)，比经纬度网格更优均匀性
- 功率归一化：每射线 P = P_tx/N_rays
- **指标**：200K 射线 × 1.25M Rx 体素 ≈ 8 min (28核 OpenMP)

### 4.3 RxHashGrid 空间哈希

- cell_size = 2 × sphere_radius → 27邻域覆盖保证
- uint64 编码：(cx<<42)|(cy<<21)|cz
- 每射线步 O(27×cell_density) 接收检测

## 五、BVH 加速结构（简历核心技能点）

### 5.1 SAH BVH 构建

```
SAH cost = (leftArea·leftCount + rightArea·rightCount) / parentArea

构建流程：
1. 16-bin 分桶：面元 centroid 沿轴线性映射到 [0,15]
2. 15 候选分裂点评估 SAH cost
3. X/Y/Z 三轴择优（取 cost 最小）
4. 小节点回退：nTris ≤ leafSize×2 → median-split
5. 自适应 leaf_size：max(4, min(32, N_faces/200))
```

- 标准 C++ 实现，无第三方依赖
- 遍历：Möller-Trumbore 射线-三角面求交（手动展开叉积）

### 5.2 场景预处理管线

```
OBJ解析 → 材质规则匹配 → Face/Edge/Wedge 拓扑构建 → BVH构建 → 场景缓存
```

- 楔边拓扑：二面角计算 → 共面/非流形/边界分类 → 楔角 = 180°-二面角
- UTD 有效楔角范围：10°~170°
- 场景缓存：二进制序列化 + FNV-1a 配置签名 → 二次运行跳过预处理

## 八、面试追问预判

### Q1：Image Method 组合爆炸怎么处理？

- uint64 签名去重（FNV-1a hash，碰撞率 ~10⁻¹¹）
- 优先级队列最佳优先（优先探索高潜力路径）
- 交互次数硬限制（max_R/T/D_count）
- 双模式分工：precise 用于小场景物理验证，SBR 用于大场景覆盖

### Q2：为什么 UTD 绕射用数值积分而非工程近似？

- Luebbers 有理近似在阴影边界过渡区误差可达 2-3 dB
- 8-point GL×16 intervals 精度 < 0.1%，代价 ~20 FLOP/调用
- 这是与商业软件（如 Wireless InSite）的关键学术差异点

### Q3：SBR 接收球半径如何定？

- cell_size = 2×sphereR（保证相邻 cell 不漏）
- 太小→漏检，太大→重复计数
- hitList 去重防止同射线重复命中
- 空间模糊固有局限：时延精度 ~sphereR/c₀

### Q4：材质参数 ε_r(f) 的频率依赖怎么处理？

- ITU-R P.2040 表格数据 → 当前线性插值
- 严格方案：幂律拟合 ε_r ∝ f^a（对标 Sionna RT `itu.py`）
- 复介电常数转换 ε_c = ε_r - jσ/(ωε₀)

### Q5：TE/TM 分解后为什么要矢量重构？

- TE ⟂ TM 正交 → 场矢量是两正交分量的矢量和
- 标量加法 A_TE+A_TM 会导致相位相反的 TE/TM 分量错误对消
- 正确做法：先分别乘 Γ_TE/Γ_TM，再矢量叠加，最后计算 |E| = √(|E_TE|²+|E_TM|²)

### Q6：OpenMP 并行中如何保证线程安全？

- 每线程独立累加器 tp[tid], th[tid]
- 每射线独立 XorShift32 种子：ri × 2654435761 + 1
- 最终归并：单线程顺序合并各线程结果
- `#pragma omp critical` 仅用于路径结果 vector push_back（O(N_paths)，非热点）

### Q7：通感一体化 (ISAC) 需要哪些信道参数？

- 通信侧：CIR, CFR, 路径损耗, RMS-DS, K-factor, AoD/AoA
- 感知侧：SINR, 检测概率, 目标 RCS, 环境散射体位置估计
- 本系统已实现：PDP/APS/CIR/信道统计/ISAC 特征集/覆盖图

### Q8：precise 和 SBR 的精度差异有多大？什么场景该用哪个？

- precise 几何路径精确到 1e-7m（交点精度），SBR 接收球引入 ~sphereR=0.15m 空间模糊
- 时延精度：precise < 0.01ns vs SBR ~0.5ns（sphereR/c₀）
- 功率精度：precise Jones 相干 vs SBR 标量 Fresnel |Γ|² 平均，差异 1-3 dB
- 场景选择：precise 用于小场景（<500 面）物理验证；SBR 用于大场景（>1000 Rx）覆盖热力图

### Q9：BVH SAH 比 median-split 好多少？16-bin 够吗？

- 理论：SAH 是最优线性扫描下的最优划分，median-split 不考虑面元分布
- 实测：SAH 比 median 少 ~15% 的射线-三角面求交次数
- 16-bin：面元 centroid 沿轴线性映射，15 个候选分裂点。对 ~10K 面场景足够；对 >100K 面可增至 32-bin
- 三轴择优：X/Y/Z 三个轴各评估 15 个候选 = 45 次 SAH cost 计算 → O(N·logN) 构建

### Q10：OBJ 格式只支持 Object 级材质，如何绑定？

- OBJ 通过 `o` 标签分 Object，每个 Object 匹配 JSON 材质规则
- 规则支持通配符（如 `Wall_*` 匹配所有墙面 Object）
- Face 结构预留了 `front_material_name` / `back_material_name` 字段，为未来逐面元材质升级做准备
- 双面材质：法线方向 = front_side（外面），对面 = back_side（内面），用于透射介质侧判定

### Q11：UTD 绕射在阴影边界 (shadow boundary) 会不会发散？

- 阴影边界上 φ-φ' = ±π → cot 参数 = ±∞，D 系数奇异
- **F(x) 过渡函数平滑**：在阴影边界上 cot 奇异 × F(x)→0 → 乘积有限
- 数值实现用 `SafeCot`（|sin| < 1e-12 时返回 ±1e12），配合 F(x→0)→0 保证数值稳定
- 这是 UTD 相对 GTD 的核心优势：GTD 在阴影边界不连续，UTD F(x) 提供平滑过渡

### Q12：为什么用 Jones 矢量而不用 Stokes/Mueller？

- Jones 矢量：保持复振幅相位 → 支持相干叠加 → precise 模式必须
- Stokes 参数：只含强度（4 实参数）→ 不支持相干叠加
- ISAC 感知场景：交叉极化鉴别率 (XPD) 需要 TE/TM 分量各自的复振幅和相位差 → Jones 唯一可行
- 代价：Jones 需 6 自由度 (3D 复矢量)，Stokes 仅需 4；但对室内通信场景 6 DoF 完全可承受

### Q13：场景缓存的失效策略是什么？

- 缓存签名 = FNV-1a hash（配置参数 + OBJ 文件哈希）
- 签名不匹配 → 缓存失效 → 重新构建
- JSON 元文件存 `cache_format_version` / `scene_source_hash`
- 已知限制：`coordinate_transform` 未纳入签名（v7 待修），`size_t` 跨 32/64 位不兼容

### Q14：为什么 SBR 用 Fibonacci 球面而非经纬度网格？

- Fibonacci 球面：黄金角度 φ = π(3-√5)，均匀分布，无极点聚集
- 经纬度网格：极点处射线密度远高于赤道 → 方向采样偏斜
- Fibonacci N 个方向近似均匀分布，角分辨率 ~√(4π/N) ≈ 1.4° @ 20K rays
- 标准实现，被学术文献和 Pathtracing 社区广泛采用

### Q15：透射 Monte Carlo 概率选择会引入多大偏差？

- 以概率 |Γ|² 选反射，以概率 T=1-|Γ|² 选透射（功率守恒）
- 期望功率：|Γ|²×ref + T×trans = |Γ|²² + T² < 1（当 |Γ|²≠0.5 时）
- **v7 H3 待修**：当前概率+缩放双重衰减导致能量损失。正确应为"概率选方向 + 功率不变"或"确定性分裂两条射线"
- 单次透射偏差约 10-20%（取决于 Fresnel 系数），多次透射指数放大

### Q16：多频点扫描怎么做？

- 当前：`frequency_hz` 是标量，需多次运行 RT.exe（每次改配置或命令行参数）
- 计划：`frequency_sweep_hz: [2.4e9, 3.5e9, 5.0e9]` → 内部循环
- 不同频率影响：Fresnel/UTD 系数（ε_c 频率相关）→ 路径集合可能不同 → 不能简单复用时延
- ITU-R P.2040 提供 1-100 GHz 标准频点

### Q17：本系统与 Sionna RT (NVIDIA) 的关键差异？

### Q18：楔边构建时非流形边 (>2 面共享) 怎么处理？

- 非流形边 (≥3 面共边) → UTD 标准公式不适用（是多个二面角的叠加）
- 当前策略：直接过滤，不构建楔边（不产生绕射）
- 用 `WedgeFlagNonManifoldSource` 标记做诊断记录
- 文献方案：对每对面分别计算二面角，逐个叠加（v8+）

### Q19：SBR 绕射的功率模型是什么？

- 三因子简化 UTD 模型（v5 DD3 已设计）：
  - 球面扩散：1/(s₁·s₂·(s₁+s₂))
  - 楔角因子：1/n²，n = (360°-α)/180°
  - Keller 锥角抑制：1/sin²β₀
- 基准校准：PEC 90° 楔 s₁=s₂=1m β₀=90° → 2%（precise UTD 验证值）
- Keller 锥 4 方向采样 (φ = 45°,135°,225°,315°) × 楔边随机采样 K 个

### Q20：如何验证 EM 计算是正确的？(L1/L2/L3)

- **L1 (解析解)**：PEC 垂直入射 (|Γ|=1, φ=180°)，Concrete 45° (TE/TM 手算对照)，Friis FSPL (λ/4πd)，Snell 折射角
- **L2 (交叉验证)**：与参考实现 (另一套独立 RT 系统) 对比路径数/功率/时延，容许偏差 < 3dB
- **L3 (统计验证)**：与 3GPP TR 38.901 InH-Office 路径损耗模型对比，RMS-DS 同量级
- 当前状态：L1 5/5 + L2 2/2 ALL PASSED

### Q21：天线方向图如何导入和插值？

- CSV 格式：(θ, φ, gain_dBi)，theta-major 排列
- 双线性插值：给定 (θ, φ) → 定位 4 个最近网格点 → 二维线性插值
- θ ∈ [0, 180°]，φ ∈ [0, 360°)，φ 边界自动 wrap
- Gain → 幅值：|A_out| = |A_in|·√(10^(gain_dBi/10))（功率比例于 |A|²）

### Q22：室内路径损耗指数与自由空间有何差异？

- 自由空间：n=2（PL ∝ d²），FSPL 标准公式
- 室内 LOS：n≈1.7-2.0（波导效应，墙面反射增强）
- 室内 NLOS：n≈3-4（多次反射透射绕射衰减叠加）
- 3GPP InH-Office：LOS PL=32.4+17.3log10(d)+20log10(f_c)；NLOS PL=38.3log10(d)+17.3+24.9log10(f_c)
- 本系统 meeting.obj 实测 PL 曲线与 3GPP 模型趋势一致

### Q23：OpenMP schedule(dynamic) vs static 如何选？

- static：预均匀分配，开销最小，但负载不均时部分线程空闲
- dynamic：运行时动态分配 chunk，负载均衡好，但有调度开销
- SBR 射线追踪：每条射线深度不固定（不同反射次数）→ 负载差异大 → dynamic 合适
- chunk_size = 1 (默认)：每次分配 1 条射线，调度频率最高，但 200K 射线时开销可忽略

### Q24：C++ 自研 Complex 类的注意事项？

- 复乘：`(a+jb)(c+jd) = (ac-bd) + j(ad+bc)`
- 复平方根：`√(a+jb) = √((r+a)/2) ± j·sign(b)·√((r-a)/2)`，r=|a+jb|
- 避免 `pow` 和 `<complex>`：自研实现不依赖标准库复数（可控精度和性能）
- NormSq() = a²+b² 用于功率计算，避免不必要的 sqrt

### Q25：这个项目最大的技术难点是什么？怎么解决的？

- **难点1（攻克）**：UTD D₁-D₄ 的符号和 cot 参数。对照 K&P 1974 Table 1 逐项验证，发现边缘坐标系对齐问题（v7 C2 修复），从基于 kOut → 基于楔面 0-face 法线
- **难点2（攻克）**：Jones 矢量 TE/TM 正交分量重构。发现标量加法导致对消（v7 C5 修复），改为矢量加法 + 归一化 Jones
- **难点3（进行中）**：precise 模式组合爆炸。当前用签名去重 + 预算限制，100K 面场景仍需候选面元空间过滤

## 九、通信算法岗高频八股 (45 题)

### 通信基础

**B1. OFDM 原理？为什么用循环前缀 (CP)？**
- OFDM：将高速数据流拆分为 N 个并行的低速子载波，各子载波正交 (Δf·T=1)，用 IFFT/FFT 高效实现调制/解调
- CP：将 OFDM 符号尾部复制到前面，长度大于最大多径时延 → 消除 ISI，将线性卷积变循环卷积 → 简化频域均衡（单抽头）

**B2. OFDM 子载波间隔与符号长度的关系？**
- Δf = 1/T_u（T_u = 有用符号长度）。Δf 越小符号越长，对 Doppler 越敏感；Δf 越大符号越短，频谱效率降低
- 5G NR 支持多种 numerology：Δf = 15·2^μ kHz (μ=0~4)，对应不同场景

**B3. OFDM 的峰均比 (PAPR) 问题及解决方案？**
- 多子载波同相加 → 峰值功率远高于平均功率 (PAPR ~ N)
- 方案：Clipping (简单但有失真)，SLM/PTS (概率降低)，DFT-s-OFDM (5G NR 上行，PAPR 接近单载波)

**B4. MIMO 的空间复用与分集增益原理？**
- 空间复用 (SM)：多天线同时发送独立数据流 → 容量 min(N_tx, N_rx) 倍
- 分集：多路独立衰落 → 合并后 SNR 更稳定 (Alamouti 空时编码)
- 复用与分集之间的 tradeoff：分集阶数 + 复用增益 ≤ min(N_tx, N_rx)

**B5. MIMO 预编码 (Precoding) 是什么？为什么要做？**
- 发射端利用 CSI 对信号做线性变换 (预编码矩阵 W)，使信号在空间中对准目标用户
- 线性预编码：MRT（最大化 SNR），ZF（消除多用户干扰），MMSE（噪声+干扰均衡）
- 非线性：DPC (Dirty Paper Coding，容量界)
- 5G NR 使用 Type I (网格波束) / Type II (线性组合波束) 码本

**B6. 信道容量公式 (Shannon) 在 MIMO 下如何扩展？**
- SISO：C = B·log₂(1+SNR)
- MIMO（CSIT 未知）：C = B·log₂det(I + (SNR/N_t)·HH^H) = B·Σ log₂(1+λ_i·SNR/N_t)
- MIMO（CSIT 已知 Water-filling）：C = B·Σ max(0, log₂(μλ_i))，μ 由功率约束确定

**B7. 大规模 MIMO (Massive MIMO) 的核心优势？**
- 天线数 N_tx ≫ 用户数 K → 信道趋于正交 (favorable propagation)
- 简单线性预编码 (MRT/ZF) 接近最优
- 阵列增益：SNR ∝ N_tx；复用增益：容量 ∝ K
- 5G NR 典型配置：64T64R @ sub-6GHz，256T256R @ mmWave

**B8. 毫米波 (mmWave) 通信的特点与挑战？**
- 优势：大带宽 (400MHz~2GHz)，短波长 → 小天线间距 → 大量天线
- 挑战：高路径损耗 (Friis 公式 f² 依赖)，绕射能力弱，易被人体/建筑遮挡
- 解决方案：波束赋形 (高增益窄波束补偿损耗)，波束管理 (初始接入/波束追踪)
- 信道稀疏性：角度域只有少数主导径 → 压缩感知信道估计

**B9. 5G NR 帧结构？**
- 10ms 无线帧 = 10 子帧 (各 1ms)，子帧 = N_slot 时隙 (取决于 numerology μ)
- 时隙 = 14 OFDM 符号 (Normal CP) 或 12 符号 (Extended CP)
- TDD 时隙格式：D (下行) / U (上行) / F (灵活)，通过 RRC 半静态或 DCI 动态配置
- 资源网格：1 RB = 12 subcarrier × 1 slot

### 信道建模

**B10. 大尺度衰落 vs 小尺度衰落的区别？**
- 大尺度 (慢变)：路径损耗 (distance-dependent) + 阴影衰落 (lognormal, σ~8dB)
- 小尺度 (快变)：多径干涉引起，通常建模为 Rayleigh (NLOS) / Rician (LOS) 分布
- 相干距离：大尺度 ~10-100m，小尺度 ~λ/2

**B11. TDL (Tapped Delay Line) 和 CDL (Clustered Delay Line) 模型？**
- TDL：时域抽头延迟线，每个抽头有固定的时延和平均功率。简单直观
- CDL：多簇模型，每簇内有 20 条子径，各子径有微小角度偏移。3GPP 38.901 标准
- CDL-A/B/C 对应不同 NLOS/LOS 场景，通过角度扩展和簇数区分
- 本系统 RT 可导出等效 TDL/CDL 参数

**B12. 3GPP TR 38.901 信道模型的关键参数？**
- 场景：RMa (农村宏站)，UMa (城市宏站)，UMi (城市微站)，InH (室内)
- 每场景定义：路径损耗模型 (LOS/NLOS)，阴影衰落 σ_SF，Rice K-factor，RMS-DS，AoA/AoD spread
- CDL 生成：给定以上参数 → 随机生成每簇角度/时延/功率 → 信道矩阵 H
- DS (delay spread) 范围：室内 ~20-50ns，城市宏站 ~300-1000ns

**B13. 什么是 Doppler 效应？相干时间如何计算？**
- Doppler 频移：f_d = (v/c₀)·f_c·cosθ（终端移动引起的频率偏移）
- 相干时间：T_c ≈ 0.423/f_d_max（信道近似不变的时间窗口）
- 5G NR 影响：高速场景 (v=500km/h @ 3.5GHz) → f_d_max ≈ 1.6kHz → T_c ≈ 0.26ms
- OFDM 子载波间干扰 (ICI)：当 f_d 接近 Δf 时显著 → 需增加 Δf 或做 ICI 消除

**B14. 空间一致性 (Spatial Consistency) 在信道建模中的意义？**
- 相邻位置的信道参数（时延、角度、功率）应平滑变化，不能完全独立随机
- 3GPP 38.901 通过空间相关随机过程 (Gaussian process) 生成空间一致的大尺度参数
- RT 天然具有空间一致性：相邻 Rx 的路径集合、时延、功率自然平滑过渡
- 这是 RT 相对统计模型的优势之一

### 信道估计与均衡

**B15. LS 和 MMSE 信道估计的原理与差异？**
- LS：Ĥ_LS = Y/X = H + N/X。简单但放大噪声，不需要信道统计信息
- MMSE：Ĥ_MMSE = R_HH(R_HH + σ²/N·I)⁻¹Ĥ_LS。最小化 MSE，需要信道自相关 R_HH 和 SNR
- MMSE 在低 SNR 下显著优于 LS（~2-5dB 增益），但复杂度 O(N³) 需对 R_HH 求逆
- 5G NR 使用 DMRS (解调参考信号) + 导频图案，支持多种导频密度

**B16. 时域均衡 vs 频域均衡？**
- 时域：线性均衡 (ZF/MMSE)，判决反馈均衡 (DFE)，MLSE (Viterbi)
- OFDM 频域均衡：每个子载波独立除以信道系数（单抽头），复杂度 O(N) vs 时域 O(N²)
- CP 消除 ISI → 频域均衡极其高效 → OFDM 核心优势

**B17. 波束管理 (Beam Management) 的流程？**
- P1 (初始波束扫描)：gNB 发 SSB burst (多方向)，UE 选最大 RSRP 波束
- P2 (gNB 侧波束精调)：gNB 在粗方向周围窄扫，UE 反馈最佳窄波束
- P3 (UE 侧波束精调)：UE 扫描接收波束，选最佳接收方向
- 5G NR 支持 64+ 波束，波束切换在 ~20ms 内完成

**B18. CSI (信道状态信息) 反馈机制？**
- UE 估计下行信道 → 选最佳 PMI (预编码矩阵)，RI (秩指示)，CQI (信道质量指示)
- 5G NR Type I CSI：基于 DFT 码本，开销低但精度有限
- Type II CSI：线性组合多个 DFT 波束 + 幅度相位量化，精度高但开销大
- CSI-RS 周期可配置 (5ms~640ms)，高精度场景 (AI/ML 增强 CSI) 为 3GPP Rel-18 热点

### 信号处理基础

**B19. Nyquist 采样定理？欠采样会怎样？**
- f_s ≥ 2·f_max（采样频率 ≥ 2 倍信号最高频率）
- 欠采样 → 频谱混叠 (aliasing)，高频分量折叠到基带，不可逆失真
- 实际系统：f_s 通常取 3-5 倍 f_max 以降低抗混叠滤波器要求

**B20. FFT 的复杂度？为什么 OFDM 用 IFFT/FFT？**
- N-point FFT：O(N·log₂N)，N-point DFT 直接计算：O(N²)
- OFDM 发射：频域符号 X[k] → IFFT → 时域信号 x[n]（多子载波叠加）
- 接收：时域信号 y[n] → FFT → 频域符号 Y[k]（子载波解调）
- 5G NR 支持多种 FFT 大小：128/256/512/1024/2048/4096

**B21. 自相关和互相关的物理意义？**
- 自相关 R_xx(τ) = E[x(t)·x*(t+τ)]：信号与自身延迟的相关程度，用于检测周期性
- 互相关 R_xy(τ) = E[x(t)·y*(t+τ)]：两信号在不同时延下的相似度，用于时间同步、匹配滤波
- 信道探测：发已知序列 → 接收信号与已知序列互相关 → 提取 CIR

**B22. 功率谱密度 (PSD) 与自相关的关系？**
- Wiener-Khinchin 定理：PSD = FFT[R_xx(τ)]（自相关的傅里叶变换）
- 带宽估计：PSD 的 3dB/6dB 带宽，或积分带宽 ∫PSD/peak

**B23. CRB (Cramér-Rao Bound) 是什么？**
- 任何无偏估计器能达到的 MSE 下限：MSE(θ̂) ≥ 1/I(θ)（I(θ) 为 Fisher 信息）
- ISAC 感知：距离估计精度 CRB ∝ 1/(SNR·BW²)，角度估计 CRB ∝ 1/(SNR·N_ant)
- CRB 是理论最优界，实际估计器 (MUSIC, ESPRIT, ML) 的性能比较基准

### 信道编码

**B24. LDPC 与 Polar 码的核心思想？5G NR 怎么用的？**
- LDPC (低密度奇偶校验)：基于稀疏校验矩阵 H，使用 Belief Propagation 迭代解码，接近 Shannon 限
- Polar 码：信道极化 → 好的 bit-channel 传信息 + 差的传冻结比特，SC/SCL 解码
- 5G NR：LDPC 用于数据信道 (eMBB)，Polar 用于控制信道
- 关键码率：LDPC 支持 1/3~8/9，Polar 支持母码 2^n (n=5~10)

**B25. HARQ (混合自动重传请求) 原理？**
- Chase Combining (CC)：重传相同版本，接收端软合并 → 累积 SNR
- Incremental Redundancy (IR)：每次重传不同编码比特 (不同的冗余版本 RV) → 等效码率降低
- 5G NR：异步 HARQ，最多 4 个 RV (0,2,3,1)，RV0 为自解码版本
- 反馈：ACK/NACK 通过 PUCCH/PUSCH，重传调度通过 PDCCH

### 链路与系统

**B26. 链路自适应 (Link Adaptation) 怎么做？**
- UE 反馈 CQI (0~15) → gNB 查表选 MCS (0~31)，对应调制阶数 + 码率
- 目标 BLER：eMBB ~10%，URLLC ~10⁻⁵
- OLLA (Outer Loop Link Adaptation)：根据实际 ACK/NACK 统计动态偏移 CQI→MCS 映射
- MCS 表：5G NR 支持 QPSK/16QAM/64QAM/256QAM，频谱效率 0.2~5.5 bps/Hz

**B27. 小区间干扰 (Inter-Cell Interference) 怎么处理？**
- 频域：ICIC（异频复用），FFR (Fractional Frequency Reuse)
- 时域：eICIC/FeICIC（ABS 几乎空白子帧，异构网）
- 空域：CoMP (Coordinated Multi-Point) JT/CB，Massive MIMO ZF 零陷
- 5G NR：灵活帧结构 + 波束协调天然抑制

**B28. SNR, SINR, RSRP, RSRQ, RSSI 的区别？**
- SNR = P_signal / P_noise（信号比噪声）
- SINR = P_signal / (P_noise + P_interference)（信号比噪声+干扰）
- RSRP = 每 RE 上 CRS/SSB 的平均接收功率（衡量覆盖强度）
- RSRQ = N_RB·RSRP / RSSI（衡量信号质量，含干扰信息）
- RSSI = 总接收功率（信号+干扰+噪声）

**B29. 链路预算 (Link Budget) 怎么算？**
- Rx Power (dBm) = Tx Power (dBm) + Tx Gain (dBi) - PL (dB) - Losses (dB) + Rx Gain (dBi)
- PL 由 Friis 公式或 3GPP 模型给出
- 接收灵敏度：取决于 BW, NF (噪声系数), SNR_min（由目标 BLER + MCS 决定）
- 余量 (Margin) = Rx Power - Sensitivity > 0 → 链路闭合

### 信道探测与测量

**B30. 信道探测 (Channel Sounding) 的基本原理？**
- 发射已知训练序列 → 接收端与已知序列做滑动相关 → 提取 CIR h(τ)
- 常用序列：Zadoff-Chu (恒定幅值 + 理想自相关)，Golay 互补序列
- 测量指标：PDP (功率时延谱)，RMS-DS，路径损耗，K-factor，AoA/AoD (阵列天线)
- 本项目的 USRP 外场经验：频谱仪测功率，信号发生器做发射源

**B31. 射线追踪 (RT) 与统计信道模型的关系？**
- 统计模型 (3GPP CDL/TDL)：参数化随机生成，速度快但缺乏特定场景的空间一致性
- RT：确定性仿真，输入场景几何+材质 → 输出路径级信道参数，天然空间一致
- 互补使用：RT 做高精度离线分析，统计模型做链路级/系统级仿真
- 混合方案：RT 做 Site-Specific 校准，统计模型做 Monte Carlo 快照

### 通感一体化 (ISAC)

**B32. ISAC (Integrated Sensing and Communication) 的核心挑战？**
- 波形设计：通信需要高谱效 (QAM)，感知需要高时延/Doppler 分辨率 (大 BW + 长 CPI)
- 资源分配：时/频/空/功率在通信与感知之间最优分配
- 互干扰：通信信号对感知是干扰，感知信号对通信也占用资源
- 硬件共享：同一套天线/射频链路 → 双工/全双工模式切换

**B33. 雷达目标检测的基本流程？**
- 发射脉冲/连续波 → 目标反射 → 接收回波 → 匹配滤波 (脉冲压缩)
- CFAR (恒虚警率) 检测：自适应阈值 (CA-CFAR, OS-CFAR)
- 距离估计：τ = 2d/c₀；速度估计：Doppler f_d = 2v/λ
- 角度估计：阵列天线波束扫描或超分辨算法 (MUSIC)

**B34. 什么是 RCS (雷达截面积)？通信 RT 中如何处理？**
- RCS = 目标散射回波功率密度与入射功率密度之比（单位：dBsm）
- 通信 RT：散射体建模为面元，Fresnel 反射/透射/绕射决定其散射特性
- 室内 ISAC：墙面 RCS 由 Fresnel |Γ|² 决定，绕射边 RCS 由 UTD D 系数决定
- 本系统通过 precise IM/UTD 获得每个散射体的复散射系数

**B35. 感知性能指标？**
- 检测概率 P_d：给定虚警率 P_fa 下正确检出概率
- 参数估计精度：CRB 下限 / RMSE 实际（距离/速度/角度）
- 分辨率：距离 Δd = c₀/(2B)，速度 Δv = λ/(2T_cpi)，角度 Δθ ≈ λ/D（D=孔径）
- ISAC 联合指标：在保证通信 QoS 下最大化 P_d 或最小化估计 RMSE

### C++ 与工程实践

**B36. C++ 中虚函数 (virtual) 和纯虚函数的区别？**
- 虚函数：父类有默认实现，子类可选 override
- 纯虚函数：`virtual void func() = 0;`，父类无实现，子类必须 override → 抽象基类
- 虚函数表 (vtable)：每个有虚函数的类维护函数指针表，运行时动态分发
- 性能影响：虚函数调用 ~2-3 次内存间接寻址；RT 内循环中避免虚函数

**B37. C++ STL 常用容器的底层实现？**
- vector：连续内存，O(1) 随机访问，尾部插入 amortized O(1)，扩容 ×1.5~2
- unordered_map：哈希表，O(1) 平均查找，链表法解决冲突
- priority_queue：二叉堆，O(log N) push/pop，O(1) top
- 本系统：priority_queue 用于 precise 最佳优先搜索，unordered_map 用于 RxHashGrid

**B38. 多线程中的数据竞争 (Data Race) 和死锁 (Deadlock)？**
- 数据竞争：两线程同时访问同一内存且至少一个是写 → 未定义行为
- 死锁：线程 A 等 B 释放锁，B 等 A 释放 → 永久阻塞
- 解决方案：mutex + lock_guard；原子操作 (atomic)；线程局部存储 (thread_local)
- 本系统：每线程独立缓冲区避免竞争，仅最后归并写一次

**B39. SIMD 向量化是什么？为什么能加速？**
- SIMD (Single Instruction Multiple Data)：一条指令同时操作多个数据
- AVX2：256-bit 寄存器，4 个 double 或 8 个 float 一次运算
- 适用场景：循环内无分支、数据连续、运算相同（Fresnel 积分点并行、向量点积）
- 本系统 D8 计划：AVX2 向量化 Fresnel 积分（对标量实现预期 2× 加速）

**B40. 内存对齐 (Memory Alignment) 的意义？**
- CPU 按 cache line (64B) 读写。未对齐 → 跨 cache line → 两次内存访问
- AVX 指令要求 32 字节对齐，AVX-512 要求 64 字节对齐
- `alignas(64)` 或 `posix_memalign` 确保对齐
- BVH 节点结构体大小 64B 对齐 → 一个 cache line 一个节点 → 最优缓存利用

### 方法论与软技能

**B41. 如何保证仿真结果可复现？**
- 固定随机数种子（SBR 每射线独立种子 ri×2654435761+1）
- 配置快照 JSON 导出（完整参数记录）
- 回归测试基线（5 配置 + Python L1/L2/L3）
- Git 版本管理 + 批次编号 (v1→v7)

**B42. 仿真与实测偏差的来源有哪些？**
- 场景建模误差：OBJ 几何不精确、未建模小物体
- 材质参数误差：ITU 表格值是典型值，非实测值
- 简化的物理模型：未包含漫散射、时变 Doppler
- 测量误差：天线方向图非理想、位置标定误差
- 本系统 L2/L3 验证：与参考实现 < 3dB 功率偏差、与 3GPP < 5ns RMS-DS 偏差

**B43. 一个算法从设计到交付的流程？**
1. 理论依据：查文献 (Balanis/K&P/Friis)
2. 公式验证：手算对照 + 边界条件检查
3. 代码实现：C++ 头文件和 .cpp 实现
4. 单元测试：解析解 L1 验证项
5. 集成测试：全配置回归
6. 性能 profiling：找热点优化
7. 文档化：开发文档 + 知识点汇总

**B44. 跨团队协作中怎么对齐接口？**
- 统一的 JSON 配置格式（所有参数一个文件）
- 明确的模块边界（六模块，每模块有独立 .h 接口）
- 输出格式标准化 (precise_paths.json, sbr_coverage.json)
- 配置快照导出 → 任何团队可以复现相同结果

**B45. 遇到性能瓶颈怎么定位和优化？**
1. Profiling 定位热点（Visual Studio Profiler / perf）
2. 分析是否算法瓶颈 (O(N²)→O(NlogN)) 或常数因子瓶颈
3. 算法优化优先：早停、缓存、降采样
4. 结合并行化：OpenMP / SIMD
5. 验证优化后结果不变（回归测试）
- 本系统案例：SBR 射线循环中 Fresnel 缓存 → 减少 80% ε_c 重复计算
