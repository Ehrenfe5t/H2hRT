# V11.5 全链路修复主计划

## 1. 目标

本轮工作不是追求“结果看起来符合预期”，而是建立可复现、可解释、可审计的天线精细化 RT 仿真链路。理论模型用于提出可证伪的预期和极限案例；若仿真偏离理论，必须定位模型假设、数值离散、实现缺陷或场景效应，不允许通过选择性删数或调参掩盖偏差。

## 2. 输入与保护边界

- 需求输入：`document/论文相关/大论文.md`、`document/v11/` 下基线、开发、配置、审查和分析文档。
- 几何保护基线：meeting 与 412 的路径数、交互类型和独立 SBR 输出。
- 代码保护边界：不改变几何路径数量的修复先在 EM、天线、配置和结果层完成；任何 SBR 行为修改均单独验证几何基线。
- 数据原则：原始输出、派生数据、图表和文字结论分层保存；所有派生指标由脚本生成，不手工抄录。

## 3. 阶段与门禁

### Phase A：语义与契约修复

目的：修复已经确认会使实验失真的实现问题。

- 天线模型只构建一次，消除逐路径 CSV 重载。
- 天线 CSV 支持并验证统一的增益/Jones 数据契约，加载失败必须显式报错。
- 路径解析以配置文件所在地为基准，实验配置不得绑定本机绝对路径。
- 明确世界坐标 Y-up 与天线局部坐标 `(right, forward, up)`，姿态扫描绕指定 up 轴生成。
- 将 SBR 采样权重传入 EM，区分采样射线路径与确定性 LOS/绕射路径。
- 区分传播 APS 与天线观测 APS；MEG 只能使用传播 APS 和实际 Rx 天线计算。
- 路径级导出补齐复振幅、采样权重和加载状态；所有写文件错误向上传播。

门禁：Release 零警告构建；解析极限案例、天线查询单测、Friis LOS、Jones 正交/匹配、phi 周期边界全部通过。

### Phase B：配置与实验体系重建

目的：使每个实验真正只改变一个自变量。

- C1：射线数收敛，比较归一化总功率、PDP、RMS-DS、K-factor、XPR 和路径集合稳定度。
- C2：按当前确定性绕射实现选择有效自变量，不再使用未接入主链的字段。
- C3：显式开关动态接收半径；分别研究固定半径敏感性与动态射线管敏感性。
- E1：同一几何路径集切换 ideal/dipole/patch/horn。
- E2：按 Y-up 世界坐标绕竖直轴扫描 Rx 姿态，并记录实际 forward/up。
- E3：固定增益与姿态，仅切换 Tx/Rx Fixed/Jones 极化。
- 412 复核：至少复现 E1/E2 的关键趋势，限制单场景泛化风险。
- E4 Coverage：仅在 P2P 主链门禁通过后启用；若当前 Coverage 接口仍冻结，则明确列为后续范围而不伪造结果。

门禁：配置快照可重放；每组实验包含目的、方法来源、控制变量、预期、判据、异常处理和运行清单。

### Phase C：仿真、统计与图表

目的：生成原始数据到论文图表的单命令流水线。

- 运行器记录版本、配置哈希、运行时间、退出码、路径数和核心指标。
- 数据质检检查 NaN/Inf、单位、路径签名一致性、功率守恒、重复记录和缺失文件。
- 统计分析同时报告效应量、相对差异、稳定区间和适用边界。
- 图表统一坐标、单位、图例、颜色、误差表达和数据来源注释。
- 每幅图配“怎么看、为什么这样、能说明什么、不能说明什么”四层解释。

门禁：分析结果可由原始输出一键重建；图表数字与表格、正文一致。

### Phase D：最终复审

- 重新构建并执行 meeting/412 基线。
- 执行解析与数值测试、快速实验、正式实验和跨场景检查。
- 对照需求追踪矩阵逐项关闭，未关闭项必须写明原因和论文表述边界。
- 最终报告同时提供零基础说明和学术定义、公式、方法、结果及局限性。

## 4. 优先级

1. P0：Bug B、Jones CSV、路径权重、传播/观测 APS、MEG。
2. P1：路径可移植性、C3 配置、姿态坐标、导出可靠性和自动化测试。
3. P2：实验重跑、统计图表、跨场景验证和论文报告。
4. P3：Coverage E4；仅在现有冻结范围允许且前序门禁通过后实施。
