# V11.1 变更日志：EM 语义校准

日期：2026-06-27
状态：✅ **已完成**
依据：`v11开发文档.md` §3.1、§3.2、§10.9

## 目标

修正路径级 EM 输出中的功率/相位语义，使 `phase_rad` 反映最终接收复电压相位，并使 `tx_power_dBm` 正确注入 EM 初始场幅度。

## 理论口径

- Tx 功率语义：方案 2 — 将 `antenna.json` 中 Tx 的 `power_dBm` 转为绝对功率尺度注入 EM 初始场，使 `power_linear` 为绝对接收功率(W)。转换公式：`P_W = 10^((dBm-30)/10)`。
- 默认 `power_dBm = 0.0`（0 dBm = 1 mW），与旧 antenna 文件兼容。
- 相位定义：`phase_rad = atan2(amplitude_imag, amplitude_real)` 等于最终接收复电压 `vr` 的相位。
- 接收功率 dBm：`power_dBm = 10*log10(power_linear * 1000)`。

### C3: `core/em/InitializeTxField.cpp` — Tx 功率注入初始场幅度
- **文件**：`core/em/InitializeTxField.cpp`，第 41-45 行
- **变更**：将 `amplitude_real = 1.0` 和 `power_linear = 1.0` 改为：
  ```cpp
  double txPowerW = std::pow(10.0, (input.tx_power_dBm - 30.0) / 10.0);
  double txAmpScale = std::sqrt(std::max(0.0, txPowerW));
  field.amplitude_real = txAmpScale; field.power_linear = txPowerW;
  ```

- **原因**：文档 §3.2 方案 2。0 dBm → 1 mW → 幅度 sqrt(0.001) ≈ 0.0316

- **原因**：`phase_rad` 此前为复矢量接收前旧值 `field.phase_rad`；新字段需填充

## 数据流完整路径（变更后）

```
antenna.json tx.power_dBm
  → V11UserConfigAdapter → TxTarget.power_dBm
  → RtPipeline BuildConfigForP2PTask → config.sbr.tx_power_dBm
  → RtRealChainRunner SolveSinglePathEM → EMSolverInput.tx_power_dBm
  → InitializeTxField: P_W = 10^((tx_power_dBm-30)/10), amp = sqrt(P_W)
  → (propagation through free space + interactions)
  → FinalizeAtReceiver:
      - vr = ComplexDot(E_inc * fsplAmp, h_rx_conj) * sqrt(rxGainLin)
      - amplitude_real = vr.re, amplitude_imag = vr.im
      - phase_rad = atan2(amplitude_imag, amplitude_real)  ← 修正点
      - power_linear = |vr|^2
      - tx_power_dBm = input.tx_power_dBm
      - power_dBm = 10*log10(power_linear * 1000)
  → ExportPaths → precise_paths.json/csv
```
