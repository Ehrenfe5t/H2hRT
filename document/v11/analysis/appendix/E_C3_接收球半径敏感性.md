## E.2 根因分析

接收球半径变化未产生任何影响，原因是 **动态射线管半径（dynamic ray tube radius）** 机制覆盖了基础半径。

当前 SBR 配置中 `enable_dynamic_rx_radius = true`（V11UserConfigAdapter 默认启用）。动态半径的计算：

```
effectiveRadius = max(rx_sphere_radius_m, 
                      pathDistance × rayTubeAngle × ray_tube_radius_scale)
effectiveRadius = clamp(effectiveRadius, 
                        max(rx_sphere_radius_m, ray_tube_min_radius_m),
                        ray_tube_max_radius_m)
```

其中：
- `ray_tube_angle_rad = 0.0` → 自动计算为 `sqrt(4π/ray_count)`
- 500k 射线 → rayTubeAngle ≈ `sqrt(4π/500000)` ≈ 0.005 rad
- 路径距离 ~3-8m → 动态半径 ≈ 3-8 × 0.005 × 1.0 ≈ 15-40 mm
- `ray_tube_min_radius_m = 0.5` → **有效半径下限为 0.5 m！**

**动态半径的最小值为 0.5 m**（`ray_tube_min_radius_m = 0.5`），远大于基础半径 0.03–0.20 m。因此基础半径的变化被动态半径完全覆盖。

## E.3 物理解释

动态射线管半径的物理意义：将每条射线视作有一定角宽度的"射线管"而非无限细的几何线。射线越远，射线管截面越大，接收球半径相应增大。这是工程补偿机制，用于减少远距离漏检。

对于 meeting 场景（路径距离 3–8 m）：
- 角宽度 ≈ 0.005 rad
- 远场射线管半径 ≈ 15–40 mm（小于 0.5 m 下限）
- 有效半径被 clamp 到 0.5 m（远大于 0.03–0.20 m 的测试范围）

## E.4 建议

要观察接收球半径的真实敏感性，有两个方案：

1. **关闭动态半径**：设置 `enable_dynamic_rx_radius = false` 或 `ray_tube_min_radius_m = 0.01`
2. **测试更大半径范围**：如 0.05 / 0.10 / 0.50 / 1.00 m，观察在动态半径生效前的基础半径影响区间

对于当前论文实验，动态半径默认开启是合理的工程选择——它保证了不同射线数下的命中一致性。
