# V11.7 大规模 Rx 与 P2P 效率专项调研

> 日期：2026-06-29  
> 状态：历史方法调研快照；其中有限电导 UTD 缺口已经关闭，后续任务状态与顺序以 `v11.7_开发实施计划.md` 为准  
> 范围：P2P 物理完整性、同 Tx 多 Rx 复用、百万至千万级三维 RadioMap、信道参数、存储与精度验收

## 1. 核心结论

1. 当前 v11.6 已经具备可信的少量 Rx 路径真值链，但仍有两个应先于性能优化关闭的物理缺口：**非 PEC 楔边仍使用 PEC UTD 系数**，以及**绕射只支持独立 `Tx-D-Rx`，不支持一阶绕射与 R/T 的混合路径**。
2. 当前 Coverage 只是对 P2P 已求得路径做轻量聚合，不是真正的广覆盖求解器。把现有外层 `for each Rx -> 完整 SBR -> 完整 EM -> 全量导出` 扩到 `1e6-1e7 Rx` 不可行。
3. v11.7 应保留两个不同产品：
   - `PreciseP2P`：少量或中等数量 Rx，输出精确路径、复系数、节点 Jones 轨迹和完整 CIR。
   - `RadioMap3D`：百万至千万级规则空间单元，一次发射、在线累积统计量，默认不保存逐路径节点。
4. 大规模空间结果不能只输出功率。根据论文的双方向全极化信道框架，最小物理描述仍是路径复系数、时延、AoD、AoA 和极化；但在大规模模式中应把这些压缩为可在线累积的功率、时延、角度和极化矩，而不是保存所有路径。
5. 对 `1e7 Rx`，首个可靠产品应是**非相干统计 RadioMap**。精确复数 `H(fc)`、CIR 和快速衰落图必须单独定义，因为“单元平均功率”和“单元中心相干复系数”不是同一个物理量。

## 2. 论文需要哪些信道量

论文第三章明确采用双方向、全极化信道描述。单条物理路径的基础量为：

- 无量纲复路径系数 `h_l`，包含振幅和相位。
- 传播时延 `tau_l` 与光程/几何长度。
- AoD 方位角、天顶角。
- AoA 方位角、天顶角。
- 路径 Jones 变换或 Rx 前的复极化场。
- LOS/NLOS、交互类型、反射/透射/绕射次数。

由这些基础量派生的论文指标包括：

- 总接收功率、路径增益/路径损耗。
- CIR、CFR、PDP。
- 最早到达时延、平均超额时延、RMS 时延扩展、相干带宽估计。
- AoA/AoD APS、平均方向、到达/离开角扩展。
- LOS 功率、NLOS 功率、Ricean K 因子或主径占比。
- co-pol/cross-pol 功率、路径级和聚合 XPR。
- 入射 APS 与 observed APS。
- 指定天线方向图、Jones 极化和姿态下的 MEG。
- 若以后加入运动：Doppler、时变 CIR 和空间一致性。

3GPP TR 38.901/ITU-R M.2412 使用的主要大尺度信道参数同样包括 DS、ASD、ASA、ZSD、ZSA、K 因子和 XPR。因此，大规模输出至少应覆盖其中与本论文相关的 DS、到达/离开角扩展、K 因子和 XPR，但不能把 3GPP 随机模型的统计分布当成确定性 RT 的硬性通过阈值。

参考：

- [论文第三章](../论文相关/大论文.md)
- [3GPP TR 38.901 V19.1.0](https://www.etsi.org/deliver/etsi_tr/138900_138999/138901/19.01.00_60/tr_138901v190100p.pdf)
- [ITU-R M.2412](https://www.itu.int/dms_pub/itu-r/opb/rep/R-REP-M.2412-2017-PDF-E.pdf)

## 3. 当前 P2P 仍缺什么

### 3.1 P0 级物理缺口

| 缺口 | 当前表现 | 风险 | v11.7 处理 |
|---|---|---|---|
| 有限电导/介质楔 UTD | `ApplyDiffractionInteraction` 未读取楔边材料，所有楔按 PEC soft/hard 系数 | 412 的玻璃、混凝土、木材绕射幅相和极化可能偏差 | 实现基于两楔面 Fresnel 反射矩阵的有限电导 UTD，并保留 PEC 极限 |
| R/T 与 D 混合路径 | 只枚举独立 `Tx-D-Rx` | 漏掉 `Tx-R-D-Rx`、`Tx-T-D-Rx` 等可能重要路径 | 支持至多一次 D、D 前后允许镜面 R/T，并用 image-space Fermat 精修 |
| 含 R/T 非线性多解 | 每个拓扑接受首个收敛 LM 解 | 非凸约束下可能漏掉同拓扑不同驻相解 | 多初值、连续延拓、按精确节点聚类保留不同解 |
| Tx/Rx 所在介质 | 根状态默认介质 0，大规模网格没有点所属介质/实体掩码 | 墙内或实体内 Rx 会污染结果 | 预计算 `valid_mask`、`medium_id`，默认排除实体内部单元 |

Sionna RT 的 UTD 实现使用有限电导楔的启发式扩展，并由两个楔面材料的 Fresnel 反射系数参与绕射矩阵；其路径求解器允许一次绕射位于镜面链中。参考：[Sionna 电磁附录 A.8](https://nvlabs.github.io/sionna/rt/tech-report/A1.html)、[Sionna Path Solver](https://nvlabs.github.io/sionna/rt/tech-report/S3.html)。

### 3.2 重要但可分阶段增强

- 真实逐频点 EM 尚未实现；当前 fixed-gain CFR 适合窄相对带宽，不能替代宽带材料和系数重算。
- 漫反射/方向性散射尚未实现，粗糙墙面和深 NLOS 区的能量丰富度可能不足。
- 多次绕射、边-面混合高阶路径未实现。
- 节点内部量为 `sqrt(W)` 功率波；任意路径位置的绝对 V/m 仍缺波前曲率/扩散张量。
- 缺少系统的 Tx/Rx 互易性、非 PEC UTD 对标、混合绕射对标和多解稳定性测试。
- 412 场景仍有 469 个退化面，源网格质量需要修复。

这些项目不都阻塞 v11.7 RadioMap MVP，但前两项 P0 缺口必须明确关闭或限制能力声明。

## 4. 当前效率为什么不能扩到 1e7 Rx

### 4.1 外层重复寻径

`RtPipeline.cpp` 当前按每个 P2P task/Rx 单独调用完整 `RunPointToPoint()`。同一个 Tx 的 Fibonacci 射线、反射/透射传播和场景求交被重复执行，空间相邻 Rx 没有复用。

### 4.2 内部数据结构按线程复制全部 Rx

`SbrEngine.cpp` 已有多 Rx 哈希网格雏形，但同时分配：

- `threadPower[thread][rx]`
- `threadHits[thread][rx]`
- `threadPaths[thread][rx]`
- `threadSeen[thread][rx]`

假设 32 线程、`1e7 Rx`，仅 `double power + int hits` 理论数据就约 3.84 GB；每个 Rx/线程的空 `vector`、`unordered_map` 对象会额外达到数十 GB，尚未包含任何路径。

### 4.3 当前 Coverage 不是 RadioMap 求解器

`CoverageEMProfile` 只是关闭复场、相位和极化输出，并对已经得到的 P2P 路径做非相干功率求和。它没有：

- 测量单元/测量平面的路径积分定义。
- 射线管立体角和单元交面积权重。
- 单次 SBR 对所有空间单元的流式沉积。
- 无偏交互采样概率补偿。
- 空间切片、checkpoint 和磁盘流式输出。

### 4.4 其他热点

- 独立绕射当前对每个 Rx 遍历全部楔边，复杂度接近 `O(N_rx * N_wedge)`。
- 确定性 R/T 双分支会随深度指数增长，不适合覆盖积分。
- CPU batch 接口目前仍是逐射线/逐 Rx fallback，且 `QueryRxHitsBatch` 只保留首个命中 Rx，不能用于密集覆盖。
- 当前有效 GPU/OptiX 后端已不在主链，只有归档代码和 SDK 样例，不能把 GPU 配置名视为已有 GPU 加速。
- 每 Rx 单独目录、JSON、CSV 的导出方式会先于计算耗尽文件系统元数据和磁盘空间。

## 5. 权威实现如何处理大规模接收区域

### 5.1 Sionna RT：单次 SBR 的 RadioMap 路径积分

Sionna 将 Path Solver 与 Radio Map Solver 明确分开：

- Path Solver 为点到点 CIR 找到离散物理路径。
- Radio Map Solver 将测量表面分成 cell，用单次 SBR 和蒙特卡洛路径积分估计每个 cell 的平均 channel gain。
- 非绕射部分一次 SBR 完成；一阶绕射另行沿楔边位置和 Keller 锥方位采样，再与主图相加。
- 计算量基本不随 cell 数增长，但 cell 越小，每 cell 命中样本越少，必须增加总样本数。
- 默认 RadioMap 是非相干路径增益和，不表示快速衰落；CIR/CFR 需要复数相干聚合。
- 支持 Russian roulette 和 gain threshold 提前终止低贡献路径。

参考：[Sionna Radio Map Solver](https://nvlabs.github.io/sionna/rt/tech-report/S4.html)。

### 5.2 Remcom Wireless InSite：APG 和精确路径

Wireless InSite X3D 使用 GPU、多线程和 exact-path 校正。对密集 Rx，其 Adjacent Path Generation：

1. 只在粗间距 Rx 上执行完整射线追踪。
2. 复用粗 Rx 路径的交互序列。
3. 为密集 Rx 恢复对应的精确路径。

这适合 P2P 批量精确 CIR，而不是取代统计 RadioMap。参考：[Wireless InSite High Fidelity Ray Tracing](https://www.remcom.com/wireless-insite-em-propagation-software/high-fidelity-ray-tracing)。

### 5.3 Altair WinProp：预处理可见性和分层输出

WinProp IRT 通过预处理数据库提前建立可见关系，使同一场景的大量位置计算更快；其输出按需选择功率、路径损耗、最小时延、时延扩展、角扩展、LOS 状态、方向性 CIR 和传播路径。参考：[WinProp Overview](https://help.altair.com/winprop/topics/winprop/user_guide/introduction/overview_winprop_c.htm)、[WinProp Result API](https://help.altair.com/Winprop/topics/winprop/user_guide/appendix/api_auto_generated/struct_win_prop___propagation___results.htm)。

### 5.4 Opal：GPU 接收球与在线归约

开源 Opal 使用 OptiX，把接收球作为加速结构并支持多 Tx/Rx 并行；默认可在 GPU 中直接累加场贡献而不保存全部 hit，需详细路径时才保留。它也指出接收球半径、重复命中和射线密度存在明显精度权衡。参考：[Opal PLOS ONE 论文](https://pmc.ncbi.nlm.nih.gov/articles/PMC8598032/)。

### 5.5 商业软件输出范围

Wireless InSite 和 WinProp 均把 dense coverage 与 full path 输出区分开。常见空间结果包括功率、路径损耗、E 场、平均到达时间、时延扩展、AoA/AoD 和角扩展；完整路径、复 CIR、MIMO H 矩阵属于更重的按需输出。参考：[Wireless InSite Outputs](https://www.remcom.com/wireless-insite-em-propagation-software/outputs)。

## 6. v11.7 推荐双求解器架构

### 6.1 PreciseP2PBatch

用途：论文基准点、少量/中等 Rx、完整 CIR、路径级解释。

```text
同一 Tx
  -> 一次 SBR 候选探索
  -> topology dictionary
  -> Rx tile 查询候选拓扑
  -> 批量 exact path / APG / 连续延拓
  -> 批量 EM
  -> 全路径或 Top-K 诊断输出
```

关键复用对象：

- Scene BVH、物理面片、楔边、可见性/PVS。
- 同 Tx 的发射方向、候选交互拓扑和 image transform。
- R/T 精修的邻近 Rx 解，作为下一个 Rx 的 warm start。
- 材料频点、角度量化的 Fresnel/UTD 系数。

### 6.2 RadioMap3D

用途：`1e6-1e7` 三维空间单元的统计覆盖和论文 E4。

```text
同一 Tx
  -> wavefront SBR（每次交互只采样一个分支并记录概率）
  -> 射线段与测量平面/网格 cell 相交
  -> 依据 ray-tube solid angle 和交面积加权
  -> 在线累积 power/delay/angle/polarization moments
  -> 独立 wedge/Keller-cone diffraction pass
  -> tile flush 到磁盘
```

RadioMap 不为每个 cell 创建物理接收球。三维规则空间建议表示为多层 XY 测量平面，或使用体素 DDA 仅遍历射线段实际穿过的 cell。第一版优先采用多层测量平面，因为其物理定义、权重和 Sionna 方法更容易对标。

## 7. 大规模 Rx 的输出分层

### L0：每 cell 必选核心图

- `valid_mask`、`medium_id`、LOS/OLOS/NLOS 状态。
- 非相干 channel gain `sum |h_l|^2`。
- 接收功率、路径损耗。
- 有效贡献数、采样数和估计方差/标准误。

### L1：论文统计图

在线维护以下充分统计量：

- `sumP`、`sumP2`。
- `sum(P*tau)`、`sum(P*tau^2)`、`minTau`。
- `P_los`、`P_nlos`、最强路径功率。
- AoA/AoD 三维单位方向的一阶、二阶加权矩。
- `P_co`、`P_cross`。

由此得到：平均时延、RMS-DS、K 因子、有效路径数、平均 AoA/AoD、角扩展和聚合 XPR。

### L2：固定天线模型库的在线观测图

论文只比较有限组 ideal/dipole/patch/horn 和姿态。每条贡献到达 cell 时，可同时对这组天线应用增益/Jones 投影，直接累积：

- 每个天线模型的 observed power/path gain。
- 每个姿态的 MEG 分子和总入射功率分母。
- observed co/cross 与 XPR。

这样不需要给 `1e7 cell` 保存完整 APS。

### L3：按需重数据

- 指定 ROI 或稀疏 anchor 的复 `H(fc)`。
- 固定 tap 数的压缩 CIR/PDP。
- Top-K 路径的 `h/tau/AoA/AoD/polarization/topology_id`。
- 完整几何节点只对 P2P anchor 导出。

## 8. 相干系数必须单独定义

非相干 RadioMap：

`G_nc = sum_l |h_l|^2`

描述 cell 平均路径增益，适合覆盖和天线模型比较，不包含快速衰落。

点中心相干系数：

`H(fc) = sum_l h_l(fc)`

描述一个确定空间点的窄带复信道，会随小于一个波长的位置变化快速起伏。不能把落在同一 cell 内不同采样点的复场直接相加后称为 cell 中心 `H`。v11.7 第一版应：

1. 全域默认输出非相干统计图。
2. 对 anchor/ROI 用 PreciseP2PBatch 输出精确复 `H/CIR`。
3. 仅在完成“命中点到 cell 中心的局部平面波重相位”和 P2P anchor NMSE 验证后，开放 dense coherent map。

## 9. 网格、半径与精度

### 9.1 网格表示

规则三维网格只保存 `origin + spacing + dimensions`，不为每个 Rx 重复保存 XYZ。用 `valid_mask` 排除墙体、家具实体和用户不关心区域。

### 9.2 分辨率

- 非相干功率/DS/角扩展图：分辨率由场景结构和统计收敛决定，不要求固定为 `lambda/2`。
- 相干快速衰落图：采样间距至少满足空间采样要求，工程上应从 `lambda/2` 或更细开始验证。
- 多分辨率策略：先粗网格定位边界和高梯度区域，再局部细化，比全域固定超细网格更合理。

### 9.3 接收半径

- PreciseP2P：接收球只发现候选，半径由射线角间隔和距离控制，最终路径必须精确连接 Rx。
- RadioMap：不再使用任意球半径；使用 cell 面积、射线管投影面积和蒙特卡洛权重。
- 若过渡版本仍使用规则 Rx 球，球半径不得超过最小网格间距的一半，并必须通过半径/射线数联合收敛证明无明显重叠偏差。

## 10. 数据结构、传递和存储

### 10.1 内存布局

- 热路径改为 Structure of Arrays，按 bounce/depth 组织 wavefront。
- cell 累加器按 tile 分块，线程只保留稀疏局部 hit buffer，随后按 `(cell_id, metric)` 排序归约。
- 累加使用 `float64/complex128`；最终 dense map 可按误差验证降为 `float32/complex64`。
- 不允许 `thread x Rx` 的完整数组和容器矩阵。

### 10.2 路径复用存储

需要保存稀疏路径时采用两表：

- `topology_dictionary`：交互类型、surface patch、wedge、介质序列。
- `rx_path_values`：`rx_id/topology_id/h/tau/AoA/AoD/Jones`。

可变长路径使用 CSR：`rx_offsets[Nrx+1] + path_values[Npath]`。Top-K 只是诊断输出；总功率和统计量必须在截断前完成累积。

### 10.3 文件格式

- 规则 dense map：优先 HDF5 chunked dataset，按空间 tile 压缩并支持切片读取、checkpoint/resume。
- 稀疏路径/统计表：Arrow RecordBatch 流或 Parquet，便于按列读取和 ZSTD/LZ4 压缩。
- CSV/JSON：仅保留 manifest、小规模摘要和调试样本。

HDF5 chunk 以单块 I/O 为单位，适合三维切片；Arrow/Parquet 是成熟列式格式，适合批量传递和只读取部分字段。参考：[HDF5 User Guide](https://docs.hdfgroup.org/archive/support/HDF5/doc1.8/UG/HDF5_Users_Guide.pdf)、[Apache Arrow](https://arrow.apache.org/)、[Parquet C++](https://arrow.apache.org/docs/cpp/parquet.html)。

### 10.4 空间预算

若核心图使用 20 个 `float32` 指标，数据量约为：

- `1e6 cell`：80 MB，另加 mask/元数据。
- `1e7 cell`：800 MB，压缩前。

若每 cell 保存 100 条路径、每条仅 64 字节，则 `1e7 cell` 已达 64 GB，尚未包含节点。因此全量路径不能作为默认广覆盖输出。

## 11. CPU/GPU 路线

1. 先完成算法和数据流，使 CPU 版具有正确的无偏估计、分块内存和确定性验证。
2. CPU 求交可评估 Intel Embree，其 AVX2/AVX-512 BVH 和 ray stream 是成熟替代方案：[Intel Embree](https://www.embree.org/)。
3. `1e7` 目标最终应预留 OptiX/CUDA wavefront 后端。OptiX 提供 RTX 硬件 BVH 求交和多 GPU 能力：[NVIDIA OptiX](https://developer.nvidia.com/rtx/ray-tracing/optix)。
4. GPU 后端只能替换求交和归约内核，不能另写一套不同物理公式；CPU 必须保留为真值对照。

## 12. 推荐验收方法

### P2P Batch 等价性

- 单 Rx batch 与 v11.6 精确 P2P 的 topology、节点、`h/tau/AoA/AoD` 一致。
- 1/10/100/1000 Rx 测试同 Tx 复用后的加速比和峰值内存。
- 互易性、材料 UTD、混合绕射和 R/T 多解专项测试通过。

### RadioMap 精度

随机和分层选择 P2P anchor，与 dense map 比较：

- 非相干路径增益误差分布。
- RMS-DS、K、XPR、AoA/AoD 平均方向和角扩展误差。
- 不同 ray sample 数的均值、方差和置信区间。
- cell 尺寸缩小时同步增加样本，验证估计方差规律。

建议工程门槛：anchor 中位功率误差小于 1 dB、90 分位小于 3 dB；RMS-DS 相对误差中位数小于 10%；主到达方向误差中位数小于 5 度。它们是 v11.7 工程验收建议，不是普适理论阈值。

### 规模和存储

- `1e6` 规则 cell 可在 CPU 分块模式完成且不随线程复制全部 cell 状态。
- `1e7` 可在固定内存预算下运行、checkpoint 和恢复；运行时间目标在首轮 profiler 基线后冻结。
- 核心 dense 输出不超过约 128 byte/cell，重数据严格按 ROI/anchor 输出。

## 13. 最终架构判断

v11.7 不应把 P2P 和覆盖模式合并成一个不断增加开关的函数。成熟路线是：

- 共用场景、材料、天线、交互物理和求交后端。
- 分离 `PreciseP2PBatch` 与 `RadioMap3D` 两个求解器。
- 用 P2P anchor 持续校准 RadioMap。
- 先保证非相干统计图可解释，再逐步开放相干 H/CIR、散射、MIMO 和时变能力。
