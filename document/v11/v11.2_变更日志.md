# V11.2 变更日志：天线方向图 phi 周期插值修正

日期：2026-06-27
状态：✅ **已完成**
依据：`v11开发文档.md` §3.4、§10.10

## 目标

修正 `AntennaPattern::QueryGainDBi()` 和 `QueryPolarization()` 中 phi 角度在 [0°, 360°) 周期边界附近的插值错误。

## 问题分析

### 根因

`AntennaPattern.h` 旧 phi 插值搜索逻辑：

```cpp
int pi = 0; for (int j = 0; j < Nphi - 1; ++j)
    if (phiDegVal >= phiDeg[j] && phiDegVal <= phiDeg[j+1]) { pi = j; break; }
```

当 phi 采样网格不覆盖整个 360° 周期（例如最后一个采样点在 355°）时：

- `phiDegVal = 359°`：位于 `[355°, 360°)` 区间，但循环只检查 `[phiDeg[j], phiDeg[j+1]]` for `j=0..Nphi-2`。**355°→360° 的间隙未被任何 j 覆盖**，`pi` 保持默认值 0。
- 结果：359° 的查询错误地使用了 phiDeg[0] 和 phiDeg[1]（例如 0° 和 5°）做插值，得到完全错误的方向图增益。

两个查询函数 `QueryGainDBi()` 和 `QueryPolarization()` 均有此缺陷。

### 受影响场景

- 天线方向图主瓣/旁瓣查询
- Jones 极化查询
- **姿态扫描中接近 0°/360° 方位的结果**（论文 E2 核心数据）
- patch/horn 等窄波束天线的旁瓣方向

## 修复方案

将 phi 区间搜索改为三段式：

1. **左周期间隙**（`phiDegVal < phiDeg.front()`）：phiLower = phiDeg.back()-360°, phiUpper = phiDeg.front()
2. **右周期间隙**（`phiDegVal >= phiDeg.back()`）：phiLower = phiDeg.back(), phiUpper = phiDeg.front()+360°
3. **正常带内**：保持原有搜索逻辑

`pFrac = (phiDegVal - phiLower) / (phiUpper - phiLower)` 统一用展开后的 phi 坐标计算。

### 数值例

phiDeg = [0, 5, 10, ..., 355]（5°步长），查询 phi=359°：

- 旧代码：pi=0（默认），pi1=1，使用 0° 和 5° 插值 → **错误**
- 新代码：进入右周期间隙，pi=Nphi-1=71，pi1=0，phiLower=355，phiUpper=365，pFrac=(359-355)/(365-355)=0.4 → 在 355°(60%权重) 和 0°≡360°(40%权重) 之间插值 ✓

## 变更清单

### C1: `QueryGainDBi()` — phi 周期插值修正
- **文件**：`core/antenna/AntennaPattern.h`，第 228-250 行
- **变更**：将旧 3 行 phi 搜索 + pFrac 替换为 19 行周期安全版本
- **旧代码**：
  ```cpp
  int pi = 0; for (int j = 0; j < Nphi - 1; ++j)
      if (phiDegVal >= phiDeg[j] && phiDegVal <= phiDeg[j+1]) { pi = j; break; }
  int pi1 = (pi + 1) % Nphi;
  double pFrac = (phiDeg[pi1] != phiDeg[pi])
      ? (phiDegVal - phiDeg[pi])/(phiDeg[pi1] - phiDeg[pi]) : 0.0;
  ```
- **新代码**：三段式判断 + 展开域 pFrac

### C2: `QueryPolarization()` — phi 周期插值修正
- **文件**：`core/antenna/AntennaPattern.h`，第 269-290 行
- **变更**：同上（与 QueryGainDBi 一致的修正）

meeting 和 412 基线均使用 `antenna_meeting.json` / `antenna_412.json`，其中 Tx/Rx 天线配置为 ideal（无 pattern_file / polarization_file）。因此 `rxAnt.pattern.loaded == false`，`QueryGainDBi()` 和 `QueryPolarization()` 不会被调用。
