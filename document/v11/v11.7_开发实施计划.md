> 修订日期：2026-06-29

## 1. 文档结论

本文件经过修订后，作为后续开发、实验设计、验收和变更追踪的主指导手册。

原计划中的有限电导 UTD 已经完成；反射/透射与绕射混合路径、多解搜索、漫反射和逐频宽带重算不再作为当前 P2P 收尾的阻塞项。后续不得因为这些研究性扩展尚未实现，就否定当前明确适用范围内的 P2P 基础链路。

当前最重要的问题已经从“继续补物理公式”转为：

1. 结果能否随射线密度和接收参数稳定收敛。
2. 各阶段真正的时间和内存热点在哪里。
3. 在不改变物理结果的前提下能优化多少。
4. 如何从单 Rx 正确扩展到同 Tx 多 Rx，而不是重复运行完整 SBR。
5. 如何把统计覆盖产品与精确 P2P 产品分开定义。

## 2. 当前已冻结的能力基线

截至本次修订，以下内容视为 P2P 基础能力基线：

- Blender/OBJ 场景中的可见表面、法向、表面材质和前后介质进入统一场景链路。
- 共面三角形缝不会成为 UTD 棱边；根据 Tx/Rx 所在外部侧筛选凸楔边。
- P2P 支持 LOS、独立镜面反射、独立透射混合径和独立 `Tx-D-Rx` 绕射径。
- 楔边两侧真实材料分别进入有限电导 UTD 复 Fresnel/Jones 计算。
- Tx 到每个交互节点再到 Rx 的复振幅、传播相位和极化演化可追踪。
- 节点内部量定义为单位 `sqrt(W)` 的归一化复路径幅度；Rx 端另行换算单路径入射 `V/m`，再经天线投影形成接收复系数。
- 每条路径的 Rx 结果仍是单路径结果；最终 CIR/CFR/信道量由多路径复数相干聚合得到。
- Release x64 构建为 0 警告、0 错误；专项自测 `34/34`，全量验证 `34/34`，`paper_ready=True`。

基线证据见：

- `v11.7_真实材料有限电导UTD修复流程.md`
- `v11.6_全链路修复与自测报告.md`
- `validation/reports/all_validation.json`
- `validation/reports/validation_summary.md`

## 3. 当前明确的能力边界

后续报告必须保留以下边界，不得把它们包装成已经支持：

- SBR 目标是发现足够多的主要贡献路径，不保证枚举数学意义上的全部路径。
- 暂不实现反射/透射与绕射混合路径。
- 暂不实现多次绕射、漫反射、方向性散射和完整时变信道。
- 暂不实现每个宽带频点重新寻径并重算材料系数。
- 楔面按局部半无限、非磁性、各向同性材料处理，不包含多层厚墙绕射模型。
- 默认 Tx/Rx 位于空气有效区域；大规模网格再单独建立实体掩码和介质归属。
- 节点 `sqrt(W)` 不是节点位置可直接测量的局部 `V/m`，也不能直接平方解释为该位置接收功率。
- 接收球只负责候选发现；最终路径精度必须由几何精修和物理残差保证。

上述边界是当前模型定义，不是立即开发清单。只有论文需求、收敛实验或外部对标证明它们形成主要误差时，才升级优先级。

## 4. 开发总原则

1. **先冻结，再优化**：没有基线结果、输入快照和验收指标，不修改核心算法。
2. **先测量，再加速**：只有 Profiler 证明的热点才进入性能重构。
3. **物理结果不因性能优化改变**：候选顺序可以变化，最终有效路径、节点、复系数和统计量必须在明确容差内一致。
4. **P2P 与 RadioMap 分离**：二者共用场景、材料、天线、交互物理和求交后端，但使用不同求解器和结果语义。
5. **候选发现与精确求解分离**：SBR/接收球发现 topology，精修连接真实 Rx，EM 只处理通过验证的最终路径。
6. **大规模模式在线归约**：默认不保存全部 hit、路径和节点，只对 anchor/ROI 输出重数据。
7. **每次新实验先调研**：先说明问题、假设、权威方法、选择理由和预期判据，再运行实验。
8. **所有近似显式记录**：相干/非相干、点值/cell 平均、采样数、方向序列、接收半径、截断、误差估计和随机种子必须进入 manifest。

## 5. 统一可追溯工作流

每个阶段均按以下顺序执行，不允许只改代码后补解释：

1. **问题定义**：当前现象、风险和本步不处理的范围。
2. **方法调研**：优先使用理论原论文、官方技术报告、成熟开源实现和商业软件公开方法。
3. **方案设计**：数据流、公式、接口、兼容策略和性能预期。
4. **验收设计**：解析测试、小规范场景、真实场景回归、性能指标和失败停止条件。
5. **最小实现**：只修改完成该目标必需的模块。
6. **分层验证**：单元测试、端到端、自变量扫描、全量回归。
7. **结果解释**：说明为什么结果应当这样变化，不能只报告“通过”。
8. **变更记录**：形成独立方案、修复日志、测试报告和输出 manifest。

建议每个阶段至少形成：

```text
document/v11/v11.7_<阶段>_方案.md
document/v11/v11.7_<阶段>_变更日志.md
document/v11/v11.7_<阶段>_验证报告.md
```

## 6. 阶段 A：P2P 基线冻结

### A0：物理链路收尾

状态：**已完成**。

完成内容包括真实材料有限电导 UTD、节点复场诊断、Rx 单路径场强与接收量、导出一致性和全量回归。

### A1：结果契约冻结

目的：防止性能优化过程中发生“文件仍能生成，但字段含义已经变化”。

实施：

- 固定路径级字段：topology、几何节点、长度、时延、AoA/AoD、复系数、功率和残差。
- 固定节点级字段：输入/输出 `sqrt(W)` 复矢量、累计时延、介质、交互系数和材料诊断。
- 固定 Rx 字段：单路径入射 `V/m`、天线投影后复系数、接收功率。
- 固定聚合字段：相干 `H/CIR/CFR` 与非相干功率和的不同定义。
- 为规范场景保存配置快照、版本号、容差和机器信息，不把临时绝对输出目录写入物理基线。

验收门槛：

- LOS、PEC 反射、介质透射、真实材料绕射、两径相长/相消场景均有黄金输出。
- 相同配置重复运行时，最终路径集合和复数结果确定性一致。
- JSON/CSV 同名物理量数值一致，单位和相干语义完整。

结果影响：不改变算法，只建立以后不可悄然破坏的契约。

## 7. 阶段 B：SBR 收敛与主要路径代表性

这是当前下一步的最高优先级实验，不应被性能重构跳过。

### 目的

证明增加射线密度后，主要路径和信道指标趋于稳定；明确当前聚类/去重结果能否代表主要贡献路径。

### 方法

对规范场景和 412 代表性 Tx/Rx 执行分层扫描：

- 射线数量至少覆盖低、中、高三个数量级，建议从 `2e4` 逐步到 `5e5` 或设备可承受上限。
- 对确定性方向序列使用多个独立旋转/偏移，避免单一 Fibonacci 方向相位造成偶然结论。
- 联合扫描接收球半径或动态半径参数，不能只增射线数而固定一个未经验证的半径。
- 聚类和严格去重分别记录原始候选数、拓扑数、精修通过数、EM 有效数和累计功率覆盖率。

观察指标：

- 总相干接收功率、非相干功率和、复 `H`。
- Top-K 主径 topology、节点位置、单径功率、时延和相位。
- RMS-DS、K 因子、AoA/AoD 主方向和角扩展、XPR。
- 新增弱径对累计功率和统计量的边际贡献。

初始工程验收门槛：

- 最高两个采样等级的总功率差原则上不超过 `1 dB`。
- 累计贡献达到 95% 的主径 topology 保留率原则上不低于 95%。
- RMS-DS、主到达方向和 XPR 的变化进入预先声明的容差。
- 接收半径改变后，经过精修的主径节点和时延保持稳定。
- 若相干功率因小相位差敏感波动，必须同时报告复 `H`、非相干功率和主径稳定性，不允许只挑稳定指标。

这些阈值是本项目的初始工程门槛，不是普适理论定律。首轮数据完成后可以收紧，但不得无记录地放宽。

失败处理：若高采样等级仍不稳定，先定位候选发现、接收球、聚类、精修或 EM 中的责任模块，不进入批量 Rx 开发。

## 8. 阶段 C：Profiler 与无语义性能优化

### C1：真实性能画像

目的：定位单 Rx 和多 Rx 的真实热点。

必须独立统计：

- 射线生成与每层 wavefront 数量。
- BVH closest-hit 和 visibility 查询次数、耗时。
- Rx 候选检测和重复命中数量。
- topology 构建、严格去重、相似性聚类和几何精修。
- EM 路径计算、节点追踪和天线查询。
- JSON/CSV 导出与总 I/O。
- 峰值 RSS、每路径字节数、每 Rx 临时对象数。

基准集至少包含 `1/10/100/1000 Rx`，并区分冷启动和重复运行。

验收：打开或关闭 Profiler 不得改变任何物理输出；报告 wall time、CPU time、峰值内存、rays/s、queries/s 和 paths/s。

### C2：第一轮安全优化

只有 C1 排名前列的热点进入本阶段。候选措施包括：

- 场景、材料频点和天线方向图只加载一次。
- 减少每射线、每候选、每路径的堆分配和字符串复制。
- 热数据改为连续数组或 SoA，调试元数据延迟生成。
- 在线统计与完整路径导出解耦，支持 `full/top_k/aggregate_only`。
- 严格去重、物理验证和昂贵精修按收益排序，避免对明显无效候选做完整 EM。
- 线程局部容器按实际命中稀疏分配，不建立 `thread x Rx` 空容器矩阵。

验收：

- 黄金场景最终 topology、节点、复系数和统计量不变。
- 每项优化分别给出前后耗时、内存和结果差异，禁止一次混入多项无法归因的重构。
- 没有稳定收益或使代码不可验证的优化应撤销或降级为实验分支。

## 9. 阶段 D：PreciseP2PBatch 同 Tx 多 Rx

只有阶段 A、B、C 通过后进入。

### 目的

保持精确 P2P 输出语义，同时避免相同 Tx 对每个 Rx 重发全部射线。

### 推荐架构

```text
同一 Tx 一次候选 SBR
  -> 射线段/交互 topology dictionary
  -> Rx 空间索引批量发现候选
  -> (rx_id, topology_id, seed) 稀疏记录
  -> 精确连接真实 Rx 并验证几何残差
  -> 每 Rx 独立 EM
  -> full / top_k / aggregate_only 输出
```

实施顺序：

1. 先完成 `1 -> 10 -> 100 -> 1000 Rx` CPU 原型。
2. 证明单 Rx batch 与现有单点 P2P 等价。
3. 再测试 `1e4` 和 `1e5 Rx` 的内存与吞吐。
4. 在未证明内存布局和候选复用有效前，不承诺 `1e6-1e7` 精确 P2P。

关键约束：

- 接收球或 Rx 网格只发现候选，不得把球面命中点当最终接收点。
- 邻近 Rx 可以复用 topology 和精修初值，但每个 Rx 的最终节点必须重新满足物理残差。
- 总功率和统计量必须在 Top-K 截断前累积。
- 峰值内存应与实际命中记录近似线性增长，不乘线程数形成完整矩阵。

验收：

- 每个 batch anchor 与独立 P2P 的最终 topology、节点、`h/tau/AoA/AoD/Jones` 在容差内一致。
- 输出 topology reuse ratio、每 Rx 候选数、精修成功率、加速比和峰值内存。
- 相邻 Rx 和稀疏 Rx 分别测试，避免只在有利密集网格上报告性能。

## 10. 阶段 E：RadioMap3D 独立产品

RadioMap 不是“把 P2P 外层循环跑一千万次”，也不是 PreciseP2PBatch 的换名。

### E1：数据模型和存储

- 规则网格保存 `origin/spacing/nx/ny/nz`，不重复保存每点坐标。
- 预计算 `valid_mask` 和 `medium_id`，默认排除实体内部。
- 按 tile 分配累加器并流式落盘，建议从多层 XY 测量平面开始。
- 核心 dense 数据使用 chunked HDF5 或等价可切片格式；JSON/CSV 只保存 manifest 和小规模样本。
- checkpoint 记录 Tx、tile 和 sample range，支持中断恢复。

### E2：第一版非相干统计求解器

首版只处理 LOS、反射和透射：

- 单次 wavefront SBR 与测量 cell/平面相交。
- 使用射线管立体角、cell 交面积和采样概率权重。
- 在线累计功率、时延、角度和极化充分统计量。
- 默认输出 cell 平均非相干增益，不冒充点中心快速衰落复信道。

每 cell 最小结果：

- `valid_mask`、介质和 LOS 状态。
- 非相干 channel gain、接收功率、路径损耗。
- 最早时延、平均时延、RMS-DS。
- K 因子或主径占比。
- AoA/AoD 平均方向和角扩展。
- co/cross 功率与 XPR。
- 有效样本数、方差或标准误。

### E3：P2P anchor 校准

- 自由空间图恢复 Friis 空间规律。
- PEC 单反射和介质板透射与密集 P2P 非相干结果对比。
- anchor 中位功率误差初始目标小于 `1 dB`，90 分位小于 `3 dB`。
- RMS-DS 相对误差中位数初始目标小于 10%，主到达方向中位误差小于 5 度。
- 样本数增加时均值、方差和置信区间符合 Monte Carlo 收敛趋势。

只有 E1-E3 通过，才把产品标记为 production RadioMap。

### E4：规模目标

- 先完成 `1e4 -> 1e5 -> 1e6 cell` 分级验证。
- `1e6 cell` 应能在 CPU 分块模式下闭环运行。
- `1e7 cell` 的首要验收是固定内存、checkpoint 和可落盘，不预先承诺运行时间。
- 核心 dense 输出初始预算不超过约 `128 byte/cell`。

## 11. 阶段 F：RadioMap 绕射与相干增强

该阶段在非绕射 RadioMap 通过后实施。

### F1：独立一阶绕射 pass

- 缓存 Tx 到凸楔的可见入射关系。
- 沿楔边位置和 Keller 锥方位采样。
- 使用与 P2P 相同的真实材料有限电导 UTD 内核。
- 与非绕射图在 cell 累加器中合并。
- 规范楔剖面必须与 P2P anchor 和权威实现对标。

### F2：ROI/anchor 相干结果

- 全域默认仍为非相干统计图。
- 指定 ROI/anchor 使用 PreciseP2PBatch 输出精确 `H(fc)`、CIR 和 Top-K 路径。
- dense coherent map 必须先解决命中点到 cell center 的重相位，并通过两径空间条纹和 P2P 复 CFR NMSE 验证。

## 12. 延后且按证据启动的物理扩展

以下项目不进入当前主线完成定义：

| 项目 | 启动条件 |
|---|---|
| `R/T...D...R/T` 混合绕射 | 论文场景对标证明独立 D 导致主要 NLOS 能量系统性缺失 |
| 同 topology 多驻相解 | 构造场景或真实结果出现首解策略导致的可重复漏径 |
| 逐频宽带重算 | 论文带宽使材料、相位或路径拓扑的窄带近似不可接受 |
| 漫反射/方向性散射 | 实测或权威对标显示深 NLOS 能量、DS 或角扩展系统性不足 |
| 多次绕射 | 一阶模型无法解释目标场景的主要覆盖区域 |
| 任意节点真实 `V/m` | 论文明确需要路径中间位置的局部物理场强，而非归一化路径状态 |

启动任何一项前，必须重新完成方法调研、方案评审和独立验收设计。

## 13. 求交后端与 GPU 路线

求交后端放在算法和数据流稳定之后：

1. Profiler 先证明 BVH 求交是主要瓶颈。
2. CPU 可评估 Embree ray stream，现有 BVH 保留为真值回归后端。
3. 后端 API 统一为 batch closest-hit、visibility 和 cell deposition。
4. 再评估 OptiX/CUDA wavefront，不恢复未经当前 CPU 真值验证的旧归档代码。
5. GPU 只替换求交、调度和归约内核，材料、Fresnel、UTD、Jones 和统计公式保持共用。

后端差异导致 face、距离、法向、路径或最终统计超出容差时，一律视为缺陷。

| G2 性能画像 | 热点、内存和 I/O 可量化，Profiler 不改结果 | 不做大规模重构 |
| G3 安全优化 | 黄金结果等价并得到可复现收益 | 回退无收益优化 |
| G4 P2PBatch | 1/10/100/1000 Rx 与独立 P2P 等价 | 不扩大 Rx 规模 |
| G5 RadioMap MVP | 解析图、P2P anchor 和 Monte Carlo 收敛通过 | 不发布覆盖结论 |
| G6 百万规模 | 分块、固定内存、checkpoint 和输出读取闭环 | 优化数据模型，不先上 GPU |

当前状态：**G0 已通过，下一目标是 G1 和 G2。**

## 15. 紧接着执行的首批任务

### 任务 1：P2P 黄金基线与结果契约

- 固定 5 类规范场景和 412 代表性 Tx/Rx。
- 建立机器可比较的路径、节点、EM 和聚合摘要。
- 明确可忽略字段和数值容差。

### 任务 2：SBR 收敛实验工具

- 自动生成射线数、方向偏移、接收半径组合。
- 自动提取主径稳定性、累计功率覆盖率和信道指标变化。
- 输出既有通俗解释又有学术统计口径的报告。

### 任务 3：分阶段 Profiler

- 为求交、Rx 查询、去重、精修、EM 和导出加入统一计时与计数器。
- 建立 1/10/100/1000 Rx 时间和内存基线。

### 任务 4：首个热点优化

- 只选择 Profiler 排名第一且不改变算法语义的热点。
- 单独提交实现、等价性验证和性能对比。

完成以上四项并通过 G1-G3 后，再启动 PreciseP2PBatch。

## 16. 本手册的完成定义

本手册不是要求在一个版本中同时完成 P2PBatch、`1e7` RadioMap、GPU 和所有物理扩展。它用于约束后续开发顺序。

v11.7 当前阶段完成定义为：

- P2P 基础物理链路和真实材料 UTD 已冻结。
- 黄金结果契约建立。
- 射线密度与接收参数收敛性得到证据支持。
- Profiler 给出可归因的时间和内存画像。
- 至少完成一轮不改变物理结果的有效性能优化。

后续里程碑分别是：

- PreciseP2PBatch：同 Tx 中等规模 Rx 的精确路径复用。
- RadioMap3D MVP：大规模非相干统计覆盖。
- RadioMap 增强：独立绕射 pass、论文统计和 ROI 相干结果。
- 后端加速：在算法稳定后接入 Embree/OptiX。

任何阶段最终都必须回答：结果是什么量、单位是什么、相干还是非相干、点值还是 cell 平均、误差如何估计、与上一基线相比改变了什么以及为什么。
