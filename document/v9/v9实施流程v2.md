# v9 实施流程 v2

本文档用于在第一轮 v9 实施后继续推进代码层面的真实闭环修复。第一轮开发已有部分进展，但复审结论是：当前代码仍未达到 v9 开发文档要求，也未达到论文实验标准。

v2 的目标不是重新规划大方向，而是针对当前代码中“局部补丁、未真实接入、验证跳过、导出缺失、语义未闭环”的问题继续收敛。

## 1. 当前复审结论

当前代码相比上一轮已有进展：

1. 新增了 `validation/run_all.py` 和部分 validation reports。
2. `TransmissionExpander.cpp` 已开始使用 `SnellRefractV2`。
3. `PathNode.h` 已增加 Snell diagnostics 字段。
4. `RtRealChainRunner.cpp` 已接入 `BuildBroadbandCFR_*`。
5. 部分 expander 已把 hard-coded `8` 替换为 `per_expander_keep_limit`。
6. `DiffractionExpander.cpp` 的非 constrained 路径开始填充绕射 diagnostics。

但当前仍不达标，主要原因：

1. `validation/run_all.py` 大量 case 是纯 Python 公式自算，没有真实验证主程序输出。
2. geometry suite 仍 skipped。
3. SBR suite 为空实现。
4. backend suite 没有真实 CPU/GPU query 对照。
5. validation runner 运行中出现 `UnicodeDecodeError`。
6. legacy `test/validate/run_all.py` 仍失败。
7. `TransmissionExpander.cpp` 未检查 `snell_residual_tol`。
8. TIR 和 Snell residual 超限没有结构化 failure reason。
9. `ExportPaths.cpp` 没有导出 Snell、medium、direction、normal、diffraction diagnostics。
10. constrained search 中 `exhaustive_debug_mode` 未完全作用于 per-state truncation。
11. `per_expander_keep_limit <= 0` 当前会保留 0 条路径，而不是“不截断”。
12. `FinalizeAtReceiver.cpp` 的 Rx Jones 匹配仍未使用真正 per-angle Rx Jones response，也未使用共轭匹配。
13. 复矢量接收分支中计算了 `rxGainLin` 但没有乘到接收复幅度。
14. `BuildBroadbandCFR_FixedGain` 仍用 `amplitude_real*cos(phase)` 重构复幅度，可能丢失 `amplitude_imag` 并重复处理相位。
15. `frequency_sweep_em` 仍不是真正逐频点 EM 重评估。
16. `ExportChannel.cpp` 未导出完整 v9 宽带产物。
17. `SceneQuery` batch 查询没有 capability-driven CPU fallback。
18. CPU fallback 的 `QueryRxHitsBatch` 返回空命中，语义错误。
19. `DiffractionExpander` constrained overload 未填 diffraction diagnostics。
20. `missing_material_policy=strict` 未真实中止材质缺失，仍 warning + vacuum fallback。
21. 未实现 `scene_preflight_report.json`。
22. `MaterialDatabase::ExtrapolateITU` 仍是启发式指数，不是 ITU-R P.2040 参数表模型。
23. SBR 缺少 per-Rx confidence、ray count convergence 和 diagnostics report。

因此，v2 的执行目标是逐项消除上述阻塞项。

## 4. 阶段 A：修复 Validation Runner 本身

### 4.1 目标

让 `validation/run_all.py` 成为可信的 v9 验证入口，而不是纯 Python 自测集合。

### 4.2 实施内容

1. 修复 subprocess 输出解码问题。
   - 所有 `subprocess.run` 必须使用：

```python
capture_output=True,
text=True,
encoding="utf-8",
errors="replace"
```

   - 或使用 bytes 读取后 `decode("utf-8", errors="replace")`。
2. validation runner 的返回码规则：
   - failed > 0，则返回非 0。
   - critical suite 出现 skipped，也应让 all suite 返回非 0，除非明确使用 `--allow-skip`。
3. `all` suite 中 geometry、wideband、backend、sbr 不允许空跑。
4. 每个 suite report 中增加：
   - `critical_skipped_count`
   - `overall_passed`
   - `paper_ready`
5. `validation_summary.md` 中必须列出：
   - failed cases。
   - skipped cases。
   - skipped 是否阻塞论文实验。
   - 真实运行的 RT.exe config。
   - 输出文件路径。
6. 纯 Python analytic case 可以保留，但必须标记为 `source=analytic_formula`。
7. 真实主程序验证 case 必须标记为 `source=rt_executable_output`。

### 4.3 自测命令

```text
python validation/run_all.py --suite smoke --backend cpu
python validation/run_all.py --suite analytic --backend cpu
python validation/run_all.py --suite all --backend cpu
```

### 4.4 验收标准

1. 不再出现 `UnicodeDecodeError`。
2. `all` suite 中如果 geometry 或 sbr skipped，`paper_ready=false`。
3. `validation_summary.md` 明确写出 skipped 是阻塞还是非阻塞。
4. runner 不再把 `20/22 passed, 2 skipped` 表述成论文可用通过。

## 6. 阶段 C：修复 Snell Residual、TIR Failure Reason 和 Top-K/Exhaustive 语义

### 6.1 目标

让 Snell 和候选控制真正符合 v9 物理和搜索语义。

### 6.2 实施内容

1. 修改 `TransmissionExpander.cpp`：
   - `SnellRefractV2` 后检查 `snell_residual_tol`。
   - residual 超限必须记录 failure reason。
   - TIR 必须记录 failure reason。
   - constrained 和 non-constrained overload 都要一致。
2. 如果 `GeometryValidityReason` 缺少合适枚举，增加：
   - `TotalInternalReflection`
   - `SnellResidualExceeded`
3. 修改 expander 截断逻辑：
   - `per_expander_keep_limit <= 0` 表示不截断。
   - `exhaustive_debug_mode=true` 表示不截断，而不是使用 `999999` 魔数。
4. 修改 `SearchEngine.cpp`：
   - full search 和 constrained search 的 per-state truncation 都必须遵守 `exhaustive_debug_mode`。
   - `per_state_keep_limit <= 0` 表示不截断。
5. validation 增加 top-K monotonic case：
   - keep=1 路径数 <= keep=8 路径数。
   - exhaustive 路径数 >= keep=8。

### 6.3 自测命令

```text

```

### 6.4 验收标准

1. residual 超限路径被拒绝并有 reason。
2. TIR 透射路径被拒绝并有 reason。
3. `per_expander_keep_limit <= 0` 不截断。
4. `exhaustive_debug_mode=true` 真正不截断。
5. constrained search 和 full search 语义一致。

## 7. 阶段 D：修复 Rx Jones、共轭匹配和 Rx Gain

### 7.1 目标

让复矢量 EM 的接收端极化和天线姿态真实生效。

### 7.2 实施内容

1. 修改 `FinalizeAtReceiver.cpp`：
   - 不再只使用 `rxAnt.polarization_vector`。
   - 若 Rx pattern 存在 polarization response，查询 per-angle Jones response。
   - 将 Rx local theta/phi 复响应转回 world basis。
   - 使用共轭匹配：

```text
vr = E_inc · conj(h_rx)
```

2. 如果 `Complex` 缺少 `Conj`，补充。
3. 复矢量分支中必须应用 Rx gain：

```text
vr *= sqrt(rxGainLin)
```

4. validation em suite 必须至少包含真实或半真实的：
   - Rx 旋转 90 度交叉极化 null。
   - 同极化接收大于交叉极化。
   - circular same-hand vs opposite-hand。

### 7.3 自测命令

```text
python validation/run_all.py --suite em --backend cpu
```

### 7.4 验收标准

1. `FinalizeAtReceiver.cpp` 使用 Rx Jones response。
2. 使用共轭匹配。
3. Rx gain 在复矢量分支中生效。
4. `em_validation.json` 中包含接收端极化和姿态验证。

## 9. 阶段 F：修复 CPU/GPU Capability Fallback 和 Backend Consistency

### 9.1 目标

### 9.2 实施内容

   - batch closest hit 需要 object filter 时，如果 GPU 不支持则 CPU fallback。
   - batch occlusion 需要 ignored face/object 语义时，如果 GPU 不支持则 CPU fallback。
   - range/all hits 如果 GPU `SupportsAllHits=false`，必须 CPU fallback。
2. `QueryRxHitsBatch` CPU fallback 不能返回空假结果。
   - 要么实现 CPU Rx grid hit 检测。
   - 要么明确返回 unsupported 并让调用方 fallback 到已有 CPU SBR 路径。
   - 不允许静默返回空命中。
3. backend suite 必须真实比较：
   - closest hit。
   - occlusion。
   - range/all hits。
   - ignored face。
   - thin slab。
4. 输出：

```text
validation/reports/backend_consistency.json
```

### 9.3 自测命令


### 9.4 验收标准
