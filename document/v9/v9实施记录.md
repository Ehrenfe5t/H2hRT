### 步骤 1：SnellRefract 数学基础增强

**日期**：2026-05-30  
**优先级**：P0  
**修改文件**：`core/common/math/Vec3.h`  
**新增文件**：`test/step1_snell_selftest.h`（自测验证）

#### 修复前的问题

1. `cosI = -Dot(incident, normal)` 未 clamp 到 `[-1,1]`。浮点误差可能导致 `cosI > 1.0`，使得 `sinT2 < 0`，进而 `sqrt(负数)` = NaN。
2. 假设调用方已确保法线指向入射侧，不自动翻转法线。背面入射时 `cosI` 为负，公式直接错误。
3. TIR 时返回零向量，无结构化诊断信息（入射角、出射角、Snell残差）。

参考依据：`v9优化开发文档.md` §4.1 第2条、§5.1.2 任务A-3。

#### 修改内容

1. **新增 `#include "MathConstants.h"`**：引入 `kPi`、`kHalfPi` 等常量，MathConstants.h 是纯 constexpr 头文件无其他依赖。

2. **新增 `SnellResult` 结构体**（`#ifndef __CUDACC__` 外部，CUDA也可用）：
   - `valid`：计算是否有效
   - `total_internal_reflection`：TIR标志
   - `direction`：折射方向向量
   - `cos_i / cos_t`：入射/出射角余弦
   - `theta_i_rad / theta_t_rad`：入射/出射角（弧度）
   - `residual`：`|n1*sin(θ_i) - n2*sin(θ_t)|`

3. **新增 `SnellRefractV2` 函数**（`#ifndef __CUDACC__` 内部）：
   - 自动翻转法线：若 `Dot(incident, normal) > 0`（背面入射），翻转法线
   - `cosI` clamp 到 `[0, 1]` 防止浮点误差
   - 结构化返回 `SnellResult`，含完整诊断

4. **保留旧 `SnellRefract`**：2个调用方（`TransmissionExpander.cpp:128, 283`）暂不改动，后续步骤迁移。

#### 自测闭环

### 步骤 26：Wedge均匀网格空间索引 (P2-2)

**日期**：2026-05-31 | **优先级**：P2 | **修改**：`Scene.h` + `WedgeAccelerationBuilder.cpp` + `SceneQuery.cpp`

**修复前**：`QueryCandidateWedges` 对全部wedge做O(W)线性扫描，大场景（10000+ wedges）每步绕射搜索都扫全表。

**修改**：
- `SceneWedgeAcceleration` 新增 `UniformGrid` 结构（AABB包围盒 + 均匀网格cell + per-cell wedge索引列表）
- `WedgeAccelerationBuilder` 构建网格：cell_size = wedge_max_distance_m/2，每个wedge插入其AABB重叠的所有cell
- `QueryCandidateWedges` 优先使用grid：定位origin所在cell → 仅搜索maxDist范围内的邻域cell → 距离筛选。grid不可用时fallback线性扫描。

**效果**：小场景行为不变；大场景绕射候选查询从O(W)降为O(1)邻域搜索。

**自测**：编译通过 + 16/16验证通过 + 管道自检7/7。
