## 第二部分：RT算法由浅入深全量讲解

### Level 1 — 物理直觉层（面试开场，3分钟）

**Q: 什么是射线追踪？为什么通信仿真要用它？**

射线追踪是一种高频近似电磁计算方法。当电磁波的波长远小于场景中的物体尺寸时（例如3.5GHz的波长约8.6cm，远小于墙壁/家具尺寸），电磁波的传播行为可以用几何光学近似——即用"射线"来描述。

核心思想：从发射天线向空间发射大量射线 → 追踪每条射线遇到障碍物后的行为（反射/透射/绕射） → 统计到达接收天线的所有射线 → 得到信道参数。

**Q: 为什么不用全波仿真（FDTD/MoM/FEM）？**

全波方法直接求解Maxwell方程组，数学上精确但对室内场景（~100m³）在3.5GHz的计算量是不可接受的——需要数TB内存和数天计算。射线追踪是高频近似下工程精度与计算效率的最佳折衷。

### Level 3 — 关键算法层（技术深度，10-15分钟）

#### 3.1 寻径引擎：优先队列最佳优先搜索

**问题**：给定Tx/Rx位置和场景几何，如何找到所有几何上可能的电波传播路径？

**算法**（逐行对应代码 SearchEngine.cpp:244-343）：

```
优先队列 PQ (按 priority = 已走距离 + 到Rx欧氏距离 - 0.5×深度 排序)

初始状态入队: {位置=Tx, 预算 R=5/T=4/D=2}

while PQ非空:
    弹出最优状态
    ├─ TryBuildLosPath: 当前点→Rx是否视线可见？→ 找到该状态的终止路径
    ├─ ExpandReflection:    镜像法 → 反射候选 (最多8个)
    ├─ ExpandTransmission:  Snell折射 → 透射候选 (最多8个)
    └─ ExpandDiffraction:   Fermat+Keller锥 → 绕射候选 (最多8个)
    
    候选去重(StateSignature) → PQ.push(所有合法候选)
```

**面试重点**——为什么不是纯Image Method？
- 纯IM只能做反射。本系统通过三展开器+优先队列实现了反射/透射/绕射的任意组合
- 每次展开后产生的新PathState重新入队，成为下一次展开的"虚拟Tx"
- 这相当于在"交互图"上做启发式搜索

#### 3.2 反射：镜像法（Image Method）

```
原理（初中物理的3D推广）：
  1. 将Rx关于候选面元平面做镜像 → MirrorRx
  2. 从Tx向MirrorRx发射射线 → 与面元平面的交点即反射点
  3. 验证：交点确实在此面元内 + Tx→交点可见 + 交点→Rx可见

代码: ReflectionExpander.cpp:158 (MirrorPointAcrossPlane)
```

#### 3.3 透射：Snell折射 + 介质切换

```
原理：
  1. Snell定律: n₁·sin(θᵢ) = n₂·sin(θₜ)
  2. 折射方向由入射方向和面法向量确定
  3. 全内反射检测: sin(θₜ) > 1 → 拒绝候选
  4. 介质切换: medium_id 从入射介质变为出射介质

代码: TransmissionExpander.cpp:113-134 (SnellRefract)
```

#### 3.4 绕射：Fermat原理 + UTD一致性绕射理论

```
几何定位阶段（DiffractionExpander.cpp）:
  1. 对每条候选楔边，用黄金分割搜索找Fermat最短路径点
     minimize over t∈[0,1]: |Tx-P(t)| + |P(t)-Rx|
  2. Keller锥验证: |cos(β₀_in)| = |cos(β₀_out)|
  3. 可见性检查

电磁计算阶段（ApplyDiffractionInteraction.cpp）:
  UTD D₁-D₄系数（Kouyoumjian & Pathak 1974）:
    Dₛₒₓₜ  = T₁+T₂-T₃-T₄  (Dirichlet/TE)
    Dₕₐᵣₔ  = T₁+T₂+T₃+T₄  (Neumann/TM)
    
    Tₙ = cot((π±(φ∓φ'))/(2n)) · F(k L a(β))
    
    F(x): Fresnel过渡函数（8点Gauss-Legendre数值积分）

  关键修正:
    - cosβ₀从出射方向正确恢复入射方向
    - L = s₁s₂sin²β₀/(s₁+s₂)（UTD标准距离参数）
    - Soft/Hard方向与系数正确配对
```

**面试重点**——为什么用8点GL而不是解析近似？
- 商业软件多用Luebbers有理近似（精度约0.5-1dB）
- 本系统用8点GL数值积分（精度<0.1dB）
- 这是论文的学术贡献点

#### 3.5 电磁计算：Fresnel公式 + Jones矢量

```
Fresnel反射系数（Balanis Ch.5）:
  Γ_TE = (cosθᵢ - √(ε_c - sin²θᵢ)) / (cosθᵢ + √(ε_c - sin²θᵢ))
  Γ_TM = (ε_c·cosθᵢ - √(ε_c - sin²θᵢ)) / (ε_c·cosθᵢ + √(ε_c - sin²θᵢ))

Jones矢量极化演算:
  1. 将入射场复投影到TE/TM基: A_TE_inc = A_inc · p_TE
  2. 乘Fresnel系数: A_TE_ref = Γ_TE · A_TE_inc
  3. 全复重构到世界坐标: pol = A_TE_ref·ê_TE + A_TM_ref·ê_TM
  4. 归一化保持|Jones|=1

复介电常数: ε_c = ε_r - jσ/(ωε₀)  (ITU-R P.2040, e^{-jωt}约定)
```

### Level 4 — 工程实现层（性能优化，5分钟）

**Q: 200万射线×125万Rx点，怎么保证性能？**

```
1. SAH-BVH空间加速（FaceBVHBuilder.cpp）:
   - 16-bin SAH (Surface Area Heuristic) 最优分裂
   - 三轴（X/Y/Z）各评估15个候选分裂点 → 选cost最小
   - cost = (leftArea×leftCount + rightArea×rightCount) / parentArea
   - 小节点(≤2×leafSize)回退到median-split

2. AVX2 SIMD（SceneQuery.cpp）:
   - 叶节点内4个三角面一批
   - Möller-Trumbore算法全向量化（__m256d）
   - ALU吞吐量×3-4

3. OpenMP多线程（SbrEngine.cpp）:
   - #pragma omp parallel for schedule(dynamic)
   - 28核自动检测，负载均衡
   - thread_local Fresnel缓存消除数据竞争

4. 材质预索引（Face.h + RtPipeline.cpp）:
   - 场景加载时预查全部面元材质
   - 运行时O(1)取值，消除hash查找+字符串拷贝+对数插值

5. 射线Morton排序（SbrEngine.cpp:216）:
   - Fibonacci球面射线按3D Morton码排序
   - 方向相近的射线连续处理 → BVH遍历缓存友好

6. 自适应终止:
   - 连续2步无新Rx命中 → 射线提前终止
```

#### Q7. 如何进行链路预算？

**标准回答** + RT项目经验：

**技术二面——突出算法深度和学术视野**：

- 主动提到已知不足（漫散射未实现、穿透深度未实现）——展示学术诚实

**综合面试——突出项目价值和团队匹配**：
