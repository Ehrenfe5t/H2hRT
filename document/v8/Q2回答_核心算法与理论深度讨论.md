## 第1问：穿透深度/透射处理 — 几何距离 vs 专门参量

### 你的理解（基本正确）

你说的对：如果一面墙由两个面元（内墙+外墙）构成，射线依次穿过这两个面元，那么：

1. 第一次透射点 P₁（进入墙体，在 front face 上）
2. 第二次透射点 P₂（离开墙体，在 back face 上）
3. 介质内传播距离 = |P₂ - P₁|（两点间的几何距离）

这个距离由 `ApplyFreeSpaceSegment` 自动计算（第1次透射到第2次透射之间的自由空间段），且此时 `current_refractive_index = n_eff`（墙体的折射率），`current_attenuation_np_per_m = α`（墙体的衰减常数）。所以**相位延迟和介质衰减已经在两透射点间的自由空间传播段中被正确计算了**。

### 为什么 v7 开发手册提到"穿透深度模型"作为待完成项？

这是因为当前实现有一个**前提假设**：墙体的 front face 和 back face 已在 OBJ 中建模为两个独立的面元，且材质绑定正确识别了它们是同一堵墙的两侧。

这个假设在以下场景中不成立：

**场景A — 单面墙（最常见的问题场景）**：
- OBJ 中墙只有一个面元（如 `.obj` 中 `f v1 v2 v3` 仅定义了外表面）
- 材质绑定为 `"Wall": {"front": "Concrete", "back": "Concrete"}`
- **透射发生时**：射线从 front 进入 Concrete → 介质变为 Concrete(n≈2.3, α≠0) → 射线在墙体介质内继续传播 → 但它永远碰不到 back face（因为 OBJ 中不存在）
- **结果**：射线在"墙体介质"中无限传播（直到功率衰减到阈值以下或被其他面元截获），而不是在正确的位置离开墙体

**场景B — 非平行双面墙**：
- 内外墙面不平行（如弧形墙、异形结构）
- |P₂ - P₁| 随入射角变化 — 这在物理上是正确的（倾斜入射 → 更长穿透距离）
- 但如果两面距离在 OBJ 中建模不准确，介质内距离就不准确

**场景C — 多层介质墙**：
- 真实墙体：石膏板 + 隔热层 + 砖 + 外饰面（4层不同介质）
- 当前模型：每层需要独立的双面建模

### "穿透深度模型"要解决的核心问题

不是"如何计算两点间距离"（这已自动完成），而是：

1. **单面墙的等效厚度**：当 OBJ 只有单面时，需要一个"名义厚度"参数 d，透射后射线沿折射方向前进 d·(1/cosθ_t) 到达虚拟的出射点，再次应用透射返回空气
2. **双面配对识别**：自动识别哪些面元属于同一堵墙的 front/back（基于距离、法向量反向、同材质等启发式规则）
3. **多层介质叠加入射**：对多层墙，逐层计算透射+介质内传播

### 结论

**你的理解在双面建模完整的场景中完全正确**——穿透距离自然由几何决定，不需要额外参数。v8 的"穿透深度模型"针对的是**单面建模场景**，此时需要一个等效厚度参数作为几何缺失的补偿。论文中可声明为"薄面元假设"（thin-surface assumption），对双面建模场景精确，对单面建模场景需引入厚度近似。

## 第2问：Precise 寻径机制 — Image Method 如何找到混合路径？

### 首先纠正一个关键误解

**当前 Precise 模式不是纯 Image Method。** 它是一个**优先队列最佳优先搜索（Priority-Queue Best-First Search）+ 三展开器混合架构**：

```
SearchEngine::Run():
  优先队列 (按 priority = 已走距离 + 到Rx距离 - 0.5×深度 排序)
    ↓ 出队一个 PathState
    ├─ TryBuildLosPath: 检查当前点→Rx 是否直连可见
    ├─ ExpandReflection:    镜像法 → 反射候选状态
    ├─ ExpandTransmission:  Snell折射 → 透射候选状态
    └─ ExpandDiffraction:   Fermat+Keller锥 → 绕射候选状态
    ↓ 候选去重 → 推入优先队列
    ↓ 循环，直到队列为空
```

### 具体回答你的问题

#### "镜像法不应该只能寻找到一次交互径吗？"

**传统镜像法**确实是：对每个面元做一次镜面反射 → 找到一次反射路径。这是标准教科书上的 Image Method。

**但本系统的 ReflectionExpander 在每一次展开时都重新做镜像法**。关键区别：

- 传统 IM：Tx → 对所有面做镜像 → 检查镜像Rx可见性 → 一次反射路径。结束。
- 本系统：Tx → 展开（反射/透射/绕射） → **新状态入队** → 从队列取出新状态 → **以新状态为"虚拟Tx"** → 再次展开（反射/透射/绕射） → ...

每次展开的 ReflectionExpander 都执行标准的镜像法（对候选面元镜像 Rx → 检查 Tx→面元→镜像Rx 的路径），但"Tx"是当前 PathState 的位置（上一次交互的命中点），不是原始 Tx。

#### "如何保证可以寻到混合径，例如 5反射+4透射+2绕射？"

搜索过程（以 R-T-D 路径为例）：

```
Step 0: 初始状态 S₀ (位置=Tx, 预算 R=5/T=4/D=2)
  → 入队

Step 1: 出队 S₀
  → TryBuildLosPath: Tx→Rx 被墙遮挡 ❌
  → ExpandReflection(S₀):   镜像法 → 找到面元 F₁ 为反射候选
    创建 S₁ (位置=P₁, 预算 R=4/T=4/D=2, has_reflection=true)
  → ExpandTransmission(S₀): Snell折射 → 找到面元 F₂ 为透射候选  
    创建 S₂ (位置=P₂, 预算 R=5/T=3/D=2, has_transmission=true, medium=墙)
  → ExpandDiffraction(S₀):  Fermat+Keller → 找到楔边 W₁ 为绕射候选
    创建 S₃ (位置=P₃, 预算 R=5/T=4/D=1, has_diffraction=true)
  → S₁, S₂, S₃ 去重后入队

Step 2: 出队 S₂ (透射状态，优先级最高)
  → TryBuildLosPath: P₂→Rx 被另一墙遮挡 ❌
  → ExpandReflection(S₂):   以 P₂ 为"虚拟Tx", 镜像法 → 找到 F₃
    创建 S₄ (位置=P₄, 预算 R=4/T=3/D=2, 已有透射+反射, mixed_path=true)
  → ExpandTransmission(S₂): 以 P₂ 为起点, Snell → 找到 F₄  
    创建 S₅ (位置=P₅, 预算 R=5/T=2/D=2, 已2次透射)
  → ...

... 以此类推，直到某状态的 TryBuildLosPath 成功（当前点→Rx 无遮挡）
```

**关键机制**：

1. **优先队列**：优先扩展距离短+深度深的路径，确保先找到低损耗路径
2. **预算倒计数**：`remaining_reflections/transmissions/diffractions` 每展开一次减1，归零后对应展开器跳过
3. **状态去重签名**：`StateSignature = hash(位置, 最后交互类型, 最后命中面, 介质, 深度, 剩余预算)`，避免同一状态重复展开
4. **路径去重签名**：`PathSignature = hash(每节点的 face_id + 交互类型 + 介质ID)`，最终路径去重
5. **BVH空间过滤**：反射展开用 AABB 扩展区域过滤候选面元，不遍历全场景面元
6. **每展开器保留Top-8候选**：按评分排序截断，防止组合爆炸

#### "是否可以严格按照配置寻找到真实的全量的几何径？"

**全量**——是的，在预算内：
- 优先队列 + 三展开器 + 去重签名 保证不遗漏任何几何有效的路径
- 验证：meeting_v3 场景（1572 路径）已验证路径数稳定

**真实**——是的，每步都有几何验证：
- 反射：面元 ID 匹配（镜像法找到的反射面元 = BVH 射线求交命中的面元）
- 透射：Snell 折射方向 + 介质切换语义 + 全反射（TIR）拒绝
- 绕射：Fermat 最短路径（黄金分割搜索）+ Keller 锥验证（|cosβ₀_in| = |cosβ₀_out|）
- 所有候选：可见性检查（Tx→交互点 和 交互点→Rx 均无遮挡）

### 与成熟实现的对比

| 特性 | 本系统 Precise | Sionna RT | Wireless InSite |
|------|-------------|-----------|----------------|
| 反射寻径 | 镜像法 | 镜像法 | SBR+IM混合 |
| 透射寻径 | Snell折射 + 介质解析 | Snell折射 | SBR射线分裂 |
| 绕射寻径 | Fermat+Keller锥+UTD | 近似公式 | UTD解析近似 |
| 混合路径 | ✅ 三展开器统一优先队列 | ❌ 仅反射IM | ✅ SBR天然支持 |
| 路径去重 | uint64签名 | 场景特定 | 未公开 |

**结论**：你的担心是合理的——如果这是纯镜像法确实找不到混合路径。但本系统是**优先队列最佳优先搜索**，镜像法仅是反射展开器使用的方法。三展开器+优先队列的组合使任意 R/T/D 序列的混合路径都能被找到，且每步都有严格的几何验证。

## 第4问：3GPP及统计模型对比 — 如何确保仿真结果准确？

### 4.1 统计模型的性质和局限

3GPP TR 38.901 等信道统计模型是**经验参数化模型**，不是第一性原理模型：

- 它们基于**大量实测数据的统计拟合** → 给出的是某个参数的**统计分布**（中位数 + 标准差），不是确定性值
- InH-Office 模型告诉你："在开放式办公室这个场景类别中，LOS路径的路径损耗指数 n≈1.73，阴影衰落 σ_SF≈3.0 dB"
- 它**不告诉你**："在坐标 (3.2, 5.7, 1.5) 处，Tx1→Rx45 的精确路径损耗是 67.3 dB"

### 4.2 正确的对比方法

**统计层面的对比（不是逐点对比）**：

```
对比流程：
  RT仿真                              3GPP TR 38.901
  ────────                             ──────────────
  场景: 120m×50m×3m 开放办公室         场景类别: InH-Office
  50 Tx位置 × 1000 Rx位置              模型: Table 7.4.3-1
  ↓                                    ↓
  每Rx提取: PL, RMS-DS, K-factor        参数: n_LOS=1.73, σ_SF=3.0dB
  ↓                                    ↓
  分LOS/NLOS                           分LOS/NLOS
  ↓                                    ↓
  拟合: PL(d) = PL(d₀) + 10n·log₁₀(d/d₀) 模型: 同上
  ↓                                    ↓
  对比: n_sim vs n_3gpp                判断: |n_sim - n_3gpp| < 某阈值 ?
```

**可对比的统计量**：

| 指标 | 3GPP InH-Office | 本系统仿真 | 对比方法 |
|------|----------------|-----------|---------|
| 路径损耗指数 n | LOS: 1.73, NLOS: 3.19 | 拟合 log(d)-PL 斜率 | 偏差 < 0.3 |
| 阴影衰落 σ_SF (dB) | LOS: 3.0, NLOS: 8.29 | PL残差的 std | 偏差 < 3 dB |
| RMS-DS μ_lgDS | LOS: -7.68, NLOS: -7.32 (lg[s]) | 所有Rx RMS-DS的均值 | 偏差 < 0.5 lg[s] |
| RMS-DS σ_lgDS | LOS: 0.22, NLOS: 0.24 | 所有Rx RMS-DS的std | 偏差 < 0.1 |
| K-factor μ (dB) | LOS: 7 | 所有LOS Rx的K均值 | 偏差 < 5 dB |
| ASA μ_lgASA | LOS: -4.19, NLOS: -4.28 | 本系统无ASA直接计算 | 需新增 |

### 4.3 验证可信度的三层体系

```
Layer 3 (统计): vs 3GPP TR 38.901
  ├─ 验证"统计意义上是否合理"
  ├─ 场景类别级对比 (InH-Office)
  └─ 确定性vs统计模型的本质差异 → 不要求精确匹配

Layer 2 (交叉): vs Sionna RT / Wireless InSite
  ├─ 验证"与其他成熟RT实现是否一致"
  ├─ 同场景同参数逐路径对比
  └─ 预期差异 < 1dB (数值实现差异)

Layer 1 (解析): vs 理论公式
  ├─ 验证"数学上是否正确"
  ├─ Fresnel |Γ|², Snell折射角, UTD D系数
  └─ 逐项对照原始文献公式
```

### 4.4 论文中的正确表述

在论文中应明确声明对比的**性质和局限**：

> "将本系统在 InH-Office 类场景中的仿真结果与 3GPP TR 38.901 统计信道模型进行统计层面对照。RT仿真基于场景几何和电磁理论的第一性原理计算，3GPP模型基于大量实测数据的统计回归。两者在本质上不同：RT仿真给出特定场景配置下的确定性结果，3GPP模型给出场景类别的统计期望。统计层面的对比用于验证RT仿真在系统级统计上是否合理，而非逐点验证。"

### 4.5 结论

**3GPP 统计模型可以作为"统计合理性"的参考**，但不能作为"逐点精度"的判据。论文的验证策略应该是：

1. L1 解析解 → 证明数学正确（这是最强证据）
2. L2 交叉验证 → 证明与其他RT实现一致（这是工程可信度）
3. L3 统计对照 → 证明在系统级统计上合理（这是论文完整性）

三层互相补充，形成完整的可信度证据链。

### 5.1 当前已实现的天线接口

struct AntennaModel {

**AntennaPattern**：CSV 双线性插值，支持任意分辨率（非固定 360×181）。

**AntennaResponse** (`AntennaModel.h:28-36`)：

```cpp
struct AntennaResponse {
    bool valid;
    double gain_linear;              // 该方向的增益 (线性)
    double gain_db;                   // 增益 (dB)
    double polarization_alignment;    // 极化匹配系数
    Vec3 effective_polarization;      // 有效极化方向
};
```

**已实现的注入链路**：

```
InitializeTxField:
  ├─ 方向图查询: direction → (θ, φ) → AntennaPattern::QueryGainDBi(θ, φ)
  ├─ 增益注入: amplitude *= √gain_linear
  ├─ 极化注入: polarization_vector = 天线极化方向
  └─ → FieldAccumulator

FinalizeAtReceiver:
  ├─ Rx方向图查询: incident_direction → (θ, φ) → QueryGainDBi
  ├─ 极化匹配: |p_tx · p_rx|²
  └─ → EMPathResult
```

### 5.3 具体缺失内容

#### A. 极化方向图（需新增）

当前 `polarization_vector` 是全空间统一的固定矢量。开题要求的"极化方向图"是**每 (θ, φ) 方向有独立的复极化响应**：

```
HFSS/CST 导出的极化方向图格式 (建议):
  θ[deg], φ[deg], Gain[dBi], Phase[deg], AxialRatio[dB], TiltAngle[deg]
  0.0, 0.0, 5.2, 0.0, 0.5, 45.0
  0.0, 1.0, 5.1, -2.3, 0.6, 44.2
  ...
```

从这些数据可重构每方向的 Jones 矢量（复极化状态），在 `InitializeTxField` 中注入 `FieldAccumulator.polarization_vector + polarization_imag`。

#### B. 天线姿态应用（需修改）

当前天线坐标系 (forward, right, up) 存储在 `AntennaModel` 中，但在 `InitializeTxField` 和 `FinalizeAtReceiver` 的方向图查询中**未做坐标旋转**：

```cpp
// 当前: 方向直接从世界坐标计算
Vec3 dir = Normalize(Subtract(rx, tx));  // 世界坐标中的发射方向

// 应为: 先转到天线局部坐标
Vec3 dirLocal = WorldToAntenna(dir, antenna.forward, antenna.right, antenna.up);
double theta = atan2(sqrt(dirLocal.x²+dirLocal.z²), dirLocal.y);  // 天线局部坐标中的角度
double phi = atan2(dirLocal.z, dirLocal.x);
double gain = antenna.pattern.QueryGainDBi(theta, phi);
```

#### C. 天线频率响应（需新增）

类似 MaterialDatabase 的多频点插值，AntennaPattern 需支持存储多频点的方向图，按仿真频率插值：

```cpp
class AntennaPattern {
    std::map<double, PatternData> patterns_by_freq; // freqHz → gain(theta, phi)
    double QueryGainDBi(double theta, double phi, double freqHz) const {
        // 对数-对数插值（同MaterialDatabase）
    }
};
```

#### D. 天线阵列（预留结构）

`AntennaArrayModel` 已经定义但未接入 `RtPipeline`：

```cpp
struct AntennaArrayModel {
    std::string array_id;
    std::vector<AntennaModel> elements;  // 每个阵元独立天线模型
    bool valid = false;
};
```

阵列仿真的标准做法：
1. 对每个阵元独立运行 IM 搜索（路径几何相同或高度相似 → 可用 synthetic array 复用）
2. 对各阵元的接收场做相干叠加（相控阵）或非相干叠加（分集天线）
3. 结果合并到 MIMO 信道矩阵

## 第6问：ISAC指标 — 从信道参数到感知特征的理论对应

### 6.1 全量信道参数：从理论到本系统实现

#### A. 大尺度信道参数（Large-Scale Parameters）

| 参数 | 定义 | 理论公式 | 本系统实现 |
|------|------|---------|----------|
| **路径损耗 PL** (Path Loss) | 发射功率与接收功率之比 | PL(dB) = P_tx - P_rx = -10log₁₀(Σ\|α_i\|²) | `BuildChannelStatistics`: `total_power_linear` → 10log₁₀(Σ\|α_i\|²) |
| **阴影衰落 SF** (Shadow Fading) | PL残差的随机波动 | SF = PL_measured - PL_fitted(d) | 需要后处理：拟合PL(d)曲线 → 计算残差std |
| **K因子** (Rician K-factor) | LOS功率 / NLOS功率 | K(dB) = 10log₁₀(P_LOS / ΣP_NLOS) | `BuildChannelStatistics`: 有 LOS 判断和 strongest_path，**需要补充计算** |
| **RMS时延扩展 DS** (Delay Spread) | 多径时延的加权标准差 | τ_rms = sqrt(ΣP_i(τ_i - τ_mean)² / ΣP_i) | 代码中有 RMS-DS 计算 (`BuildChannelStatistics`)，但当前实现较基础 — **需要确认** |
| **角度扩展 ASA/ASD** (Angular Spread) | AoA/AoD的加权标准差 | φ_rms = sqrt(ΣP_i(φ_i - φ_mean)² / ΣP_i) | `EMPathResult` 有 aoa_theta_deg/phi_deg, aod_theta_deg/phi_deg — **缺统计聚合** |
| **交叉极化比 XPR** (Cross-Polarization Ratio) | 同极化/交叉极化功率比 | XPR = 10log₁₀(P_co / P_cross) | `FieldAccumulator` 有完整 Jones 矢量 — **需新增统计** |

#### B. 小尺度信道参数（Small-Scale Parameters）

| 参数 | 定义 | 理论公式 | 本系统实现 |
|------|------|---------|----------|
| **CIR** (Channel Impulse Response) | 时延域复信道响应 | h(τ) = Σα_i·δ(τ-τ_i)·e^(jφ_i) | `BuildCIR`: ✅ 复数振幅(tap.amplitude_real + j·tap.amplitude_imag) |
| **PDP** (Power Delay Profile) | 时延域功率分布 | P(τ) = Σ\|α_i\|²·δ(τ-τ_i) | `BuildPDP`: ✅ (delay, power_dB) 散点 |
| **APS** (Angular Power Spectrum) | 角度域功率分布 | P(θ,φ) = ΣP_i·δ(θ-θ_i, φ-φ_i) | `BuildAPS`: ✅ 改用 aoa_theta_deg |
| **CTF** (Channel Transfer Function) | 频域响应 | H(f) = FFT{h(τ)} | ❌ 未实现 — CIR → FFT → CTF |
| **CSI** (Channel State Information) | = CTF 的离散采样 | H[k] = H(f_k), k=0..N_subcarrier-1 | ❌ 未实现 — 依赖 CTF |

#### C. ISAC 专用特征（感知侧）

| ISAC 特征 | 物理含义 | 从本系统可计算的字段 | 实现状态 |
|-----------|---------|---------------------|---------|
| **路径数** | 多径丰富度 | `EMPathResultSet.valid_result_count` | ✅ `BuildISACFeatureSet` |
| **最早到达时延** | ToF测距精度上界 | `min(delay_s)` | ✅ `BuildISACFeatureSet` |
| **最强路径功率** | 目标检测SNR | `max(power_linear)` | ✅ `BuildISACFeatureSet` |
| **平均极化幅度** | 去极化效应强度 | `mean(polarization_magnitude)` | ✅ `BuildISACFeatureSet` |

当前 ISAC 前沿研究关注的特征（2023-2025 文献）：

**本系统在 ISAC 学术前沿的独特贡献点**：
1. **全极化信道建模**：Jones 矢量 TE/TM 相干提供完整的极化散射矩阵 → 比其他RT工具（Sionna仅圆极化基，Wireless InSite未公开极化处理）更精细
2. **混合路径 ISAC**：反射+透射+绕射任意组合 → 可仿真"穿墙感知"（through-wall sensing）场景
3. **逐路径电场**：复振幅+相位+极化 → 支持相干感知（相位干涉检测），这在其他RT仿真工具中不常见
