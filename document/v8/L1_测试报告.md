## 8. 修复记录 #1: 极化退化bug

### 问题

`ApplyReflectionInteraction` / `ApplyTransmissionInteraction` / `ApplyDiffractionInteraction` 中的TE/TM基向量构建。

当入射方向与面法向量平行（正常入射）时，`Cross(kInc, n) = (0,0,0)`。回退代码：
```cpp
if (Length(eTE) < 1e-9) eTE = MakeVec3(1.0, 0.0, 0.0);
```
当 kInc=(1,0,0) 时，回退的 eTE 与 kInc **共线** → `Cross(eTE, kInc)` 再次退化 → eTM=(0,0,0) → pTE=0, pTM=0 → 场振幅=0。

### 修复

三个 `Apply*Interaction.cpp` 文件中的回退逻辑改为：
```cpp
if (Length(eTE) < 1e-9) {
    if (std::fabs(kInc.x) < 0.9)
        eTE = Normalize(Cross(kInc, MakeVec3(1.0, 0.0, 0.0)));
    else
        eTE = Normalize(Cross(kInc, MakeVec3(0.0, 1.0, 0.0)));
}
```
选择与kInc不共线的参考方向，确保Cross结果始终是kInc的有效正交方向。

### 影响范围

- **L1-04/06**：修复前 power_linear=0（正常入射透射） → 修复后恢复正确值
- **已有场景**：L1-02/03 非正常入射，不受影响
- **ApplyDiffractionInteraction**：同时修复了 eSoft_dir 和 ex 的退化回退
- **不改变已有仿真结果**（仅在原本退化的场景下生效）


## 9. 修复记录 #2: 绕射可见性查询

### 问题

`DiffractionExpander` 检查 Tx→绕射点 和 绕射点→Rx 的可见性时，`VisibilityQueryContext` 未忽略楔边的两个面。楔面自身阻挡了所有通向绕射点的射线路径。

### 修复

1. `VisibilityQueryContext` 新增 `ignored_face_id2` 字段
2. `FaceQueryContext` 新增 `ignored_face_id2` 字段
3. `AcceptFace()` 增加第二忽略面检查
4. `DiffractionExpander` 在可见性检查前设置双面忽略：
```cpp
vc.ignored_face_id = cand.positive_face_id;
vc.ignored_face_id2 = cand.negative_face_id;
```

### 结果

可见性检查修复后，绕射候选不应再被楔面自身阻挡。但L1-08测试中绕射路径(type=6)仍未命中。进一步分析表明根因可能在：
- 楔边构建阶段的法向量方向影响楔角计算
- 两面法向量方向不正确可能导致楔边被拒绝（如 `diffractable=false`）
- 需要追踪 `WedgeBuilder` 中的 `wedge.diffractable` 赋值逻辑


## 10. L1-07: Brewster角入射扫描 — 分析

### 当前状态

仿真可以运行并找到反射路径，但功率对比受偏振效率影响。

### 技术分析

默认天线极化 `polarization_vector = (0,1,0)`（Y-up线偏振）。在不同入射角下：

- eTE方向 = Cross(kInc, n)：对斜入射主要沿Z方向
- eTM方向 = Cross(eTE, kInc)：在XY平面内

Y-up偏振在不同入射角下的TE/TM投影比例不同：
- 正常入射(kInc沿X)：Y-up = 纯TE (cosθ=0°时)
- 斜入射(kInc偏Z)：Y-up = TE+TM混合

因此仿真 `power_linear / FSPL_power` 不等于纯|Γ_TE|²也不等于纯|Γ_TM|²，而是两者的偏振加权组合。

### 解决方案

要纯化入射偏振：
- **方案A**：设置 `polarization_file` 指定不同入射角的偏振方向（需实现CSV偏振方向图加载）
- **方案B**：为每个入射角调整场景几何使默认Y-up恰好对准纯TE或纯TM方向
- **方案C**：接受当前混合偏振，在对比时计算偏振加权后的理论|Γ|²值

当前推荐方案C（零代码改动），在论文中说明"偏振匹配效率已内禀包含在仿真功率中"。

## 12. L1-09: 极化旋转Jones矢量 — 分析

### 目标

验证反射/透射/绕射后的Jones矢量包含了正确的TE/TM混合信息——
即入射45°线偏振经介质斜反射后变为椭圆偏振。

### 当前限制

`AntennaModel` 的 `Ideal` 模式硬编码了 `polarization_vector = (0,1,0)`（Y-up）。要设置自定义偏振方向需要：
1. 实现 `polarization_file` CSV加载（已预留字段）
2. 或在 `AntennaFactory::BuildTxAntennaModel` 中增加 `polarization_vector` 参数

### 仿真数据中已有的极化信息

`EMPathResult` JSON 导出已包含：
```json
"polarization_vector": [rx, ry, rz],   // Jones矢量实部
"polarization_imag":    [ix, iy, iz]    // Jones矢量虚部
```

当前默认Y-up偏振的反射/透射路径中，`polarization_vector` 和 `polarization_imag` 已正确记录了TE/TM混合后的复极化状态。**验证Jones矢量的基础设施已就绪**，缺的是多样化的入射偏振配置能力。
