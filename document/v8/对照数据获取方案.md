## 二、L1 解析解 — 最可靠、最优先

### 2.1 为什么解析解是最强验证

解析解基于 Maxwell 方程组在特定边界条件下的精确解，**数学上绝对正确**。如果你的代码在解析解场景下结果不一致 → 代码有bug。这比任何交叉验证都更可靠。

### 2.2 建议的规范测试场景集（6个标准场景）

#### 场景 L1-1：自由空间 LOS（已完成，验证FSPL）

```
场景: 空场景 (无任何面元)
Tx: (0, 1, 1), Rx: (3, 1, 1)
频率: 2.4 GHz

理论预测:
  - 路径数: 1 (LOS)
  - PL = FSPL = 20log₁₀(4πd/λ) 
    = 20log₁₀(4π×3/0.125) = 20log₁₀(301.6) ≈ 49.6 dB
  - delay = d/c = 3/3×10⁸ = 10 ns
  - AoA = (θ=90°, φ=0°) [水平方向]
  - AoD = (θ=90°, φ=0°)

对照源: Friis公式 (1946)
代码验证点: InitializeTxField → ApplyFreeSpaceSegment → FinalizeAtReceiver
```

#### 场景 L1-2：单面PEC反射（验证反射角+Fresnel |Γ|²=1）

```
场景: 一面无限大PEC墙 (σ→∞)
  - 位置: x=2平面, 法向量(-1, 0, 0)
  - 尺寸: 100m×100m (模拟无限大)
  - 材质: PEC (ε_r=1, σ=1e10) 或直接绑定为理想导体
Tx: (0, 1, 1), Rx: (3, 1.5, 1)

理论预测:
  - 反射点: (2, 1.25, 1) [Tx和Rx的中间位置对称映射]
  - θ_i = θ_r (入射角=反射角验证)
  - |Γ_TE|² = 1, |Γ_TM|² = 1 (理想导体全反射)
  - 路径数: 2 (LOS + 一次反射)
  - 反射路径相位: 180°翻转 (PEC边界条件)

对照源: Balanis Ch.5; Born & Wolf §1.5
代码验证点: ReflectionExpander + ApplyReflectionInteraction
```

#### 场景 L1-3：单面介质反射（验证Brewster角）

```
场景: 一面介电质墙 (ε_r=5.24, σ=0 — 无损Concrete模型)
  - 位置: x=2平面, 法向量(-1, 0, 0)
Tx: (0, 0.5, 1), Rx: (3, 0.5, 1) → θ_i扫描 0°~89°

理论预测:
  - Brewster角: θ_B = arctan(√ε_r) = arctan(√5.24) ≈ 66.2°
    → TM反射系数在Brewster角处为零
  - |Γ_TM|² = 0 @ θ_i = 66.2°
  - 功率守恒: |Γ|² + (n₂cosθ_t/n₁cosθ_i)·|τ|² = 1

对照源: Balanis Ch.5 Eq.(5-49)/(5-50); Born & Wolf §1.5.2
代码验证点: ApplyReflectionInteraction::FresnelTM
建议新增L1验证项: L1-8 Brewster角验证
```

#### 场景 L1-4：单面介质透射（验证Snell+透射功率守恒）

```
场景: 一面双面建模介质墙 (ε_r=5.24, σ=0)
  - 位置: x=2平面, 双面完整 (front_medium=空气, back_medium=墙体内部)
Tx: (0, 0.5, 1), Rx: (3, 0.5, 1) → 透射方向经Snell折射

理论预测:
  - Snell: n₁sin(θ_i) = n₂sin(θ_t)
    sin(θ_t) = sin(θ_i)/√5.24
    例如 θ_i=45°: sin(θ_t) = 0.707/2.289 = 0.309 → θ_t ≈ 18.0°
  - τ_TE = 2cosθ_i/(cosθ_i+√(ε_c-sin²θ_i))
  - τ_TM = 2√ε_c·cosθ_i/(ε_c·cosθ_i+√(ε_c-sin²θ_i))
  - 功率守恒验证

对照源: Balanis Ch.5; Born & Wolf §1.5.2
代码验证点: TransmissionExpander + ApplyTransmissionInteraction
建议新增L1验证项: L1-6 TM透射功率守恒
```

#### 场景 L1-5：单楔PEC绕射（验证UTD D₁-D₄）

```
场景: 一个PEC楔 (90°楔角)
  - 棱边: 从(2, 0, -10)到(2, 0, +10), 垂直方向
  - 0-face: 面元 x=2, y<0 (水平面)
  - n-face: 面元 x>2, y=0 (垂直面)
  - 楔角 α=90° → n=(360°-α)/180°=1.5
Tx: (0, 1, 0.5), Rx: (4, -1, 0.5)

理论预测:
  - 绕射点: Fermat最短路径 (黄金分割搜索验证)
  - Keller锥: |cos(β₀_in)| = |cos(β₀_out)|
  - D_soft/hard vs K&P (1974) Table 1 数值基准
  - 扫描楔角α=30°~150° → 验证绕射损耗vs楔角曲线

对照源: Kouyoumjian & Pathak (1974); Balanis Ch.12
代码验证点: DiffractionExpander + ApplyDiffractionInteraction
建议新增L1验证项: L1-9 UTD绕射vs楔角扫描
```

#### 场景 L1-6：双面穿墙透射（验证介质内传播）

```
场景: 一面双面建模混凝土墙
  - Front face: x=2平面, 法向量(-1,0,0), front=空气, back=Concrete
  - Back face: x=2.3平面, 法向量(1,0,0), front=Concrete, back=空气
  - 墙厚: 0.3m
  - 材质: Concrete (ε_r=5.24, σ=0.05 S/m @3.5GHz)
Tx: (0, 1, 1), Rx: (4, 1, 1)

理论预测:
  - 第一次透射(T1): 空气→Concrete, Snell + Fresnel
  - 介质内传播: d_wall/cos(θ_t) = 0.3/cos(θ_t) 米
  - α = k₀·Im(√ε_c) ≈ 0.15 Np/m (损耗介质衰减)
  - 第二次透射(T2): Concrete→空气, Snell + Fresnel
  - 总路径 = 两个自由空间段 + 一个介质内段

对照源: 多层介质Fresnel (Born & Wolf §1.6); ITU-R P.2040
代码验证点: 连续两次TransmissionExpander + ApplyTransmissionInteraction + ApplyFreeSpaceSegment (介质折射率)
```

### 2.3 L1场景集实现方式

**策略A：手写OBJ文件**（推荐，精确可控）

OBJ格式极其简单，可直接手写：

```obj
# L1-2: 单面PEC反射墙
v  0 -50 -50    # 顶点1
v  0  50 -50    # 顶点2
v  0  50  50    # 顶点3
v  0 -50  50    # 顶点4
vn -1 0 0       # 法向量
f 1//1 2//1 3//1
f 1//1 3//1 4//1
```

**策略B：Python脚本生成OBJ**（推荐，批量生成不同参数场景）

```python
def generate_wall_scene(filename, x_pos, width, height, eps_r, sigma):
    """生成单面墙OBJ场景"""
    hw = width/2; hh = height/2
    with open(filename, 'w') as f:
        f.write(f"v {x_pos} {-hw} {-hh}\n")
        f.write(f"v {x_pos} {hw} {-hh}\n")
        f.write(f"v {x_pos} {hw} {hh}\n")
        f.write(f"v {x_pos} {-hw} {hh}\n")
        f.write(f"vn {-1 if x_pos > 0 else 1} 0 0\n")
        f.write("f 1//1 2//1 3//1\n")
        f.write("f 1//1 3//1 4//1\n")
```

### 3.2 Sionna RT 交叉验证具体方案

#### Step 1: 安装与验证

```bash
pip install sionna
# Sionna RT需要TensorFlow，但IM模式只需numpy
```

#### Step 2: 创建等价场景

Sionna RT 使用 Mitsuba XML 或自己的 Scene API：

```python
import sionna
from sionna.rt import Scene, Transmitter, Receiver, PlanarArray

# 创建等价于L1-3的场景: 一面介质墙
scene = Scene()
# 添加墙面...
scene.add(sionna.rt.Rectangle(
    "wall", 
    [0, 0, 0],    # 左下角
    [0, 10, 0],   # 右下角 (10m宽)
    [0, 0, 10],   # 左上角 (10m高)
    material="itu_concrete"
))
scene.frequency = 3.5e9  # 3.5 GHz

# 设置Tx/Rx (与本系统坐标完全一致)
tx = Transmitter("tx", position=[1, 1, 1])
rx = Receiver("rx", position=[3, 1, 1])
scene.add(tx); scene.add(rx)

# 运行IM
paths = scene.compute_paths(max_depth=5, diffraction=False)
```

#### Step 3: 逐路径对比指标

| 本系统导出字段 | Sionna导出字段 | 容差 | 备注 |
|--------------|---------------|------|------|
| `delay_s` | `paths.tau` | < 0.1 ns | 时延 |
| `aoa_theta_deg` | `paths.theta_t` | < 0.5° | 到达仰角 |
| `aoa_phi_deg` | `paths.phi_t` | < 0.5° | 到达方位角 |
| `aod_theta_deg` | `paths.theta_r` | < 0.5° | 离开仰角 |
| `aod_phi_deg` | `paths.phi_r` | < 0.5° | 离开方位角 |
| `power_linear` | `paths.a²` | < 1 dB | 路径增益 |
| valid路径数 | len(paths) | < 10% | 路径数量 |

#### Step 4: 差异分析

```python
# 路径匹配: 基于delay+AoA聚类
def match_paths(my_paths, sionna_paths):
    matched = []
    for mp in my_paths:
        best = min(sionna_paths, key=lambda sp: 
            abs(sp.tau - mp.delay_s)*1e9 + 
            abs(sp.theta_t - mp.aoa_theta_deg)*0.1)
        if abs(best.tau - mp.delay_s)*1e9 < 0.2:  # 0.2ns窗口
            matched.append((mp, best))
    return matched
```

### 3.3 Sionna RT 的限制（需在论文中声明）

1. **绕射差异**：Sionna用近似公式 vs 本系统UTD 8点GL积分 → 绕射路径预期有2-3dB差异
2. **极化坐标系**：Sionna用圆极化基 vs 本系统TE/TM线极化基 → 物理等价但数值表达不同
3. **透射处理**：Sionna的透射建模方式需对比确认
4. **材质插值**：Sionna也基于ITU-R P.2040，但插值方法可能不同
5. **Sionna版本**：需锁定特定版本号，未来版本可能改变算法

## 四、L3 统计模型对照

### 4.1 3GPP TR 38.901 InH-Office

3GPP为室内热点场景定义了完整的信道统计模型（TR 38.901 v16.1.0 §7.4）：

**可用作对照的LSP参数：**

| 参数 | InH-Office LOS | InH-Office NLOS |
|------|---------------|-----------------|
| 时延扩展 μ_lgDS | -7.68 log10(s) | -7.32 log10(s) |
| 时延扩展 σ_lgDS | 0.22 | 0.24 |
| AoA扩展 μ_lgASA | -4.19 | -4.28 |
| AoD扩展 μ_lgASD | -4.60 | -4.64 |
| K因子 μ_K | 7 dB | N/A |
| 阴影衰落 σ_SF | 3.0 dB | 8.29 dB |
| PL指数 n | 1.73 | 3.19 |

**对照方法**：
```
本系统仿真 → 50个随机Tx位置 × 1000个Rx位置
→ 分LOS/NLOS → 拟合PL指数 → 计算DS/ASA/ASD/K因子统计
→ 与3GPP参数对比 → 偏差在容差范围内即为通过
```

**重要声明**：3GPP参数是统计期望值，不是逐点精确预测。偏差<0.3 DS单位或<2dB SF标准差即为一致。

### 4.2 ITU-R P.1238 室内传播模型

ITU-R P.1238 提供简单室内场景的路径损耗经验公式：
```
PL(d) = 20log₁₀(f) + N·log₁₀(d) + L_f(n) - 28
```
其中 N=20-30（办公室场景），L_f(n)为楼层穿透损耗。

**适用场景**：本系统仿真结果应与N≈20-30（LOS office）范围一致。

## 六、自洽性检验 — 零成本、永久有效

### 6.1 互易性（Reciprocity）

电磁波传播满足互易性：Tx↔Rx 互换，信道相同。

```
验证方法:
  配置1: Tx=(1,1,1), Rx=(3,1,1)
  配置2: Tx=(3,1,1), Rx=(1,1,1)
  → 两条配置的路径集（delay, power, AoA/AoD互换）应完全一致
```

### 6.2 射线数收敛性（SBR）

```
验证方法:
  同一场景，SBR射线数从1万→5万→20万→50万→100万→200万
  → active_rx_count 趋于稳定
  → 每Rx的power_dBm趋于稳定
  → 计算相对标准差 → 随射线数增加而减小
```

### 6.3 路径数稳定性（Precise）

```
验证方法:
  同一场景，改变max_path_depth从2→3→4
  → 新增更深路径的总功率贡献递减
  → 路径损耗趋于稳定
```

### 6.4 BVH正确性自检

```
验证方法:
  随机抽取100条射线，分别用:
    A) BVH遍历 → 求交结果
    B) 暴力遍历全场景三角面 → 求交结果
  → A和B求交的face_id和distance完全一致
```

## 八、最重要的建议：先做L1场景集

**L1解析解场景集是投入产出比最高的验证手段。** 理由：

1. **数学确定性**：解析解是Maxwell方程组的精确解，不是近似的、不是统计的、不是另一个软件的输出
2. **零外部依赖**：不需要装Sionna、不需要买Wireless InSite、不需要做实验
3. **可复现**：任何人拿OBJ手写文件+配置JSON就能复现，论文中可提供
4. **覆盖全链路**：6个场景从FSPL→反射→透射→绕射→双面穿墙→Brewster角，覆盖所有核心算法
5. **发现bug能力强**：L1场景下1dB偏差就意味着代码有bug

**v8立即行动项**：
- 在 `test/scenes/L1_canonical/` 下建立6个手写OBJ场景
- 在 `test/reference/L1_expected.json` 中存储每个场景的理论预期值
- 每次代码修改后跑 `test/validate/validate_L1.py` 做回归

---

*然后在此基础上，Phase 1引入Sionna RT做L2交叉验证（中等会议室场景），Phase 2用3GPP做L3统计验证（InH-Office标准场景），形成完整的三层验证数据基线。*
